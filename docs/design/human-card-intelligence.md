# 人卡智能架构 — 五层模型

> 设计定稿。基于 Gibson 可供性、SDT 自决理论、BDI 意图架构、GPS 反向规划。
> 动物只有标签层+行为管线。人卡多出五层。

---

## 一、理论根基

### 1.1 Gibson 可供性 → 标签不是行为，是行动可能

标签不是"我会猎"，是"世界对我提供了猎这个行动可能"。`capability.hunt` 不描述狼——它描述环境和狼之间的关系。同一个环境对不同标签集合的生物提供不同的可供性。狼看到的是"可猎的鹿"，鹿看到的是"可吃的草"，人看到的是"可敲的石头"。

### 1.2 SDT 自决理论 → 需求只有三个

自主（我选的吗）、胜任（我做得来吗）、关联（我和营地/火/他人有连接吗）。`fire_bond` 和 `hunger` 不是需求——它们是需求的满足物。火灭了→关联受威胁→高优先级中断。饿了→胜任受威胁（徒手猎兔胜率低）。三需求不分层级，并行检测。

### 1.3 BDI 意图架构 → 欲望和意图是两种东西

欲望池里有很多东西——想吃、想加柴、想建菇棚。它们每轮竞选，但菇棚永远输给吃和柴。意图槽里的东西不竞选——一旦被提升为意图，它获得保证的时间配比，直到完成或条件失效。

### 1.4 GPS 反向规划 → 从目标往回查前置条件

不是"先做什么再做什么"的顺序列表。而是从终点往回走——建菇棚需要什么？木构件×2。木构件需要什么？木头×3+加工能力。每一步检查前置，不满足就生成子目标，不可达就放弃整个目标。造了十节车厢不会忘了火车头。

### 1.5 Leontiev 活动三层 → 动机·动作·操作

狩猎这个活动，驱动它的动机是"食物"。实现它的动作是"追猎物"。动作拆成操作——走路、举矛、刺。动机层决定为什么做，动作层决定做什么，操作层自动执行。

### 1.6 Bourdieu 惯习 → 标签集合就是惯习

"持久的、可转移的倾向系统，作为被结构的结构，倾向于发挥结构化结构的功能——即作为实践和表象的生成和组织原则，这些实践和表象客观上可以被管制和规则化，却无论如何不是遵守规则的产物。"每个词都在描述我们的标签系统。

## 二、五层架构

```
┌─────────────────────────────────────────────────────┐
│ 第五层 · 意图引擎（BDI + GPS 反向规划）               │
│                                                     │
│ 信念 = 世界状态（草地上有鹿、手上有石头、火在烧）       │
│ 欲望池 = 候选目标（猎兔、制刀、建菇棚）                 │
│ 意图槽 = 已承诺的长期目标（建菇棚）                    │
│                                                     │
│ 长期意图一旦激活，获得固定时间配比（默认 30%）           │
│ 短期欲望每轮 GPS 反向检测前置条件                      │
│ 意图不参与竞选，只在条件失效或完成时释放                │
├─────────────────────────────────────────────────────┤
│ 第四层 · 执行承诺                                     │
│                                                     │
│ "我能不能在灾难发生之前把手头的事做完？"                │
│                                                     │
│ 灾难检测：物理威胁（狼<3格）> 生存威胁（火灭/饿死）> 便利│
│ 阈值计算：当前任务剩余时间 < 灾难到达时间 → 不中断        │
│ 阈值计算：当前任务剩余时间 > 灾难到达时间 → 紧急放弃      │
│                                                     │
│ 只有紧急威胁才能打断正在执行的动作                      │
│ 非紧急中断不采纳，先把手头的事干完                       │
├─────────────────────────────────────────────────────┤
│ 第三层 · SDT 需求检测                                 │
│                                                     │
│ 自主：我的行动是被迫的还是我选的？                      │
│   狼在附近 → 被迫逃跑 → 自主受威胁 → 高优先级           │
│   安全 + 火 + 食物 → 我在主动建造 → 自主满足            │
│                                                     │
│ 胜任：我能有效完成吗？                                 │
│   徒手猎兔胜率低 → 胜任低下 → 制刀优先                  │
│   石刀入手 → 胜任恢复 → 可以猎兔了                      │
│                                                     │
│ 关联：我和营地/火/他人有连接吗？                        │
│   火灭了 → 关联严重受威胁 → 最高优先级                  │
│   火在烧 + 有棚 → 关联稳固                             │
│                                                     │
│ 饥饿不是需求——它是胜任的干扰信号。火不是需求——它是关联    │
│ 的满足物。三个需求平行检测，不分层级。                   │
├─────────────────────────────────────────────────────┤
│ 第二层 · 可供性检测（Gibson）                          │
│                                                     │
│ 环境对标签集合提供了什么行动可能？                       │
│                                                     │
│ 手持石头 + 地上有石头 → 可制石刀                        │
│ 手持石刀 + 鹿在草地上 → 可猎鹿（成功率提升）             │
│ 火在烧 + 有兔肉 → 可烤肉                               │
│ 自由时间 + 木头×3 + 有制作能力 → 可建菇棚               │
│                                                     │
│ 可供性不是物体属性，是标签×世界的产物。                   │
│ 环境不变，动物不同，可供性完全不同。                      │
├─────────────────────────────────────────────────────┤
│ 第一层 · 标签层（已有的 CARD_CAPABILITIES + CardDB）    │
│                                                     │
│ 标签定义生态位。不同标签组合 = 不同动物。                │
│ 人类标签：omnivore, tool_dependent, fire_bond,         │
│          opportunistic, 以及后天获得的 artisan...     │
└─────────────────────────────────────────────────────┘
```

## 三、一个完整决策循环

```
1. 可供性层：世界对当前的标签集合提供了什么？
   → 可制石刀、可猎兔、可采浆果、可喂火、可收集木头

2. 需求层：三个根需求的当前状态
   → 自主：安全（无狼）    胜任：低下（无工具）    关联：稳固（有火有棚）

3. 意图引擎：
   欲望池 → 制石刀（胜任↑）、猎兔（胜任+饥饿↓）、收集燃料（关联维持）
   意图槽 → 空（没有长期目标在运行）

4. GPS 反向规划：制石刀
   查前置：石头×2 → 手上有 + 地上有 → 满足
   查前置：有锤击能力 → capability.use_tool → 满足
   → 可行

5. 执行承诺：当前无紧急威胁，承诺执行制石刀

6. 行为管线：走过去 → 捡石头 → 敲击 → 石刀完成

7. 新状态：石刀入手。胜任恢复。新可供性出现：可猎兔（有效率）。
   下一轮重复。
```

## 四、长期意图的完整生命周期

```
激活条件：生存盈余（三项需求全部满足）+ 环境存在超越生存的可供性
         + GPS 反向规划确认前置全满足

激活：欲望"建菇棚"→ 提升到意图槽
时间配比：30% 给意图，70% 给生存

执行中：
  正常工作（分配的时间片内推进子目标）
  ↓
  紧急中断（狼来了 → 100% 时间逃跑，意图保留不丢弃）
  ↓
  火灭了（关联严重受威胁 → 100% 时间取火，意图保留不丢弃）
  ↓
  前置复查（木头被闪电劈了 → GPS 检测到前置失效）
  ↓
  不可达 → 释放意图槽 → 意图槽空出
  可达 → 继续推进

完成：建菇棚完成 → 释放意图槽 → 新的长期目标候选重新扫描
```

## 五、和动物的区别

| | 动物（狼） | 人（玩家卡） |
|---|---|---|
| 标签层 | ✅ 相同 | ✅ 相同 |
| 可供性层 | ✅ 相同 | ✅ 相同 |
| 需求层 | ❌ 无 | ✅ SDT 三需求 |
| 意图层 | ❌ 无（欲望=意图，直接执行） | ✅ BDI 意图槽 |
| 执行承诺 | ❌ 无（紧急中断即可） | ✅ 承诺阈值 |
| 行为管线 | ✅ 相同 | ✅ 相同 |
| 长期目标 | ❌ 无 | ✅ 时间配比 |
| 空闲 | ❌ 不会空闲（饥饿就猎） | ✅ 全满足时合法 idle |

动物的"思考"在标签层结束。标签组合 + 环境可供性 = 直接触发行为。人多了三层——需求评估、意图与欲望分离、执行承诺。

## 六、和现存游戏 AI 的核心差异

| | RimWorld | The Sims | DF | 我们 |
|---|---|---|---|---|
| 决策方式 | 优先级树 | 效用评分 | 需求队列 | BDI+GPS |
| 长期目标 | ❌ | ❌ | ❌ | ✅ 意图槽 |
| 振荡防护 | 硬编码顺序 | 随机选 Top4 | 无 | ✅ 执行承诺 |
| 模块化 | JobGiver 脚本 | 智能体广播 | 硬编码 | ✅ 标签组合 |
| 空闲 | 无（总有事做） | 娱乐活动 | 发呆 | ✅ 合法 idle |
