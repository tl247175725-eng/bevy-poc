## 1. 草压力（参数调优，架构不动）

架构已经标签化：`terrain.riverbank` → `RIPARIAN_GRASS_INTERVAL` → `_update_riparian_grass()` → `LIVING_GRASS_CAP`。不改逻辑，只调参数值。

| 常量 | 现值 | 新值 | 效果 |
|------|------|------|------|
| `LIVING_GRASS_CAP` | 18 | **12** | 全图活草上限收紧，草更频繁接近紧缺 |
| `RIPARIAN_GRASS_INTERVAL` | 6.0s | **9.0s** | 恢复变慢，消耗高于恢复时草会下降 |

预期：常态草量从 14~18 降到 8~12，动物更频繁遇到"无草可吃"，触发移动和竞争。

## 2. 捕猎风险（标签扩展：escape.* + hunt_success_chance）

### 2.1 现状

`WorldRules.hunt_target_score()` 只回答"选谁做目标"，不判断攻击是否成功。猎物有 `capability.flee` 但仅在逃跑行为中使用，从未进入捕猎判断。

### 2.2 新增逃逸能力标签

归入 **capability** 维度：

| 标签 | 含义 | 分配给 |
|------|------|--------|
| `capability.escape_fast` | 高速奔跑，低成功率 | deer, fox |
| `capability.escape_small` | 体型小，难命中 | rabbit, fieldMouse |
| `capability.escape_cover` | 在灌木/草丛中更难被抓 | fox, fieldMouse |

羊不加 escape 标签——它是驯化动物，逃逸能力弱，是狼的基础猎物。

### 2.3 新增 WorldRules 查询

```gdscript
WorldRules.hunt_success_chance(hunter: CardBase, target: CardBase) -> float
```

输入：捕猎者能力标签 + 猎物 escape 标签 + 距离 + 地形  
输出：0.0 ~ 1.0 的成功概率

规则：
- 基础成功率 = 0.85
- 猎物有 `escape_fast`：-0.40
- 猎物有 `escape_small`：-0.25
- 猎物有 `escape_cover` 且在 cover 格上：-0.35
- 猎物眩晕中：+0.50
- 距离 > 1：每格 -0.05（追逐消耗体力）
- 结果钳制在 0.10 ~ 0.95

### 2.4 接入点

行为四段式的 **execution** 阶段：`best_hunt_target` 选出目标后，捕猎者的行为 tick 调用 `hunt_success_chance`，失败则猎物安全、捕猎者进入 huntCooldown。

**不变量**：玩家/狼/狐狸全走同一套 `hunt_success_chance`。捕猎者的差异在于 target 选择逻辑（已有 `hunt_target_score`），不在成功率计算公式内。

## 3. 移动速度标签化（能力驱动替代 card_type 查表）

### 3.1 现状

`game_types.gd` 的 `ACTOR_MOVE_EVERY` 按 `card_type` 字符串查表，每次新增卡都要手动加一行。

### 3.2 新增移动能力标签

归入 **capability** 维度：

| 标签 | 间隔 | 分配给 |
|------|------|--------|
| `capability.move_fast` | 0.50s | fieldMouse |
| `capability.move_quick` | 0.55s | rabbit, fox, wolf |
| `capability.move_normal` | 0.75s | sheep, deer |
| `capability.move_slow` | 0.80s | lamb, deerFawn, fieldMousePup |
| `capability.move_juvenile` | 1.20s | foxCub, wolfCub |

已有标签 `capability.move` 保留，作为最低门槛（"能不能动"），新增标签表达"多快"。

### 3.3 新增 WorldRules 查询 + 重构 game_types

```gdscript
WorldRules.move_interval_for(actor: CardBase) -> float
```

按优先级匹配：`move_fast` > `move_quick` > `move_normal` > `move_slow` > `move_juvenile` > `move`。全部不匹配时返回默认 0.70。

`game_types.move_interval()` 改为转发 `WorldRules.move_interval_for()`，`ACTOR_MOVE_EVERY` 表可逐步移除。

### 3.4 灌木/草丛加速通用化

当前 `FOX_COVER_MOVE_BONUS = 1.35` 只适用于狐狸。改为通用规则：

`WorldRules.cover_move_multiplier(actor)`：有 `capability.use_cover` 且当前在 `cover.small` 格上 → 返回 1.35，否则 1.0。

### 3.5 玩家/mushroomFarmer/traveler

这三个不属于生态动物，行为节奏由任务系统控制，暂不参与此次标签化。
