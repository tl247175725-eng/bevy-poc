# v3.0-A 修正切片：封闭生态调参 + 鹿繁殖 + 桃源人可见远观

> 状态：待实现（已按 2026-05-30 反馈修订：桃源是封闭生态，不靠地图外迁入/迁出补平衡）  
> 类型：A2/A3 验收修正，不进入 v3.0-B  
> 原因：当前鹿更像“肉更多的羊”；生态初始数量/资源恢复/繁殖/捕食尚未调成自洽循环；桃源三卡已注册但 live 不生成，F5 看不到。

---

## 1. 问题判断

### 1.1 鹿的问题

`deer` 已接入吃草、逃跑、被狼/玩家捕猎、尸体/鹿肉/熟肉链，但它的差异主要体现在：

- `largePrey`
- 肉量更高
- 狼评分更优先

这不足以让玩家感到“鹿是一个生态角色”。如果鹿只是更特别的羊，它不会支撑后续生态链，也不会给桃源人观察、玩家狩猎、资源压力带来新的判断。

更重要的是：桃花源不是有外部入口的普通野外地图，本轮不能靠“动物从地图外迁入 / 离开地图”伪造生态平衡。羊、兔、狼、鹿、草、树、浆果等都应被视为同一个封闭生态里的成员。

生态调试的目标不是让每一轮都静止不变，而是让系统在合理波动中不崩溃：

- 草地资源会被吃掉，也会在适合 terrain 上恢复。
- 食草动物会繁殖，也会被捕食。
- 狼会捕食，但食物足够时不无限猎杀。
- 玩家干预会改变数量结构。
- 长时间 trace 中不应出现草地清零、食草动物清零、狼无限杀戮、肉/尸体爆炸。

### 1.2 桃源人的问题

三种桃源人已进入：

- `CardDB`
- `WorldRules` capability
- `ecosystem_behavior_key`
- 避让距离查询

但当前 live 初始世界没有生成 `taoyuanElder` / `taoyuanForager` / `taoyuanYouth`。`social_observation_trace` 也只验证定义和规则函数，不验证世界里真的出现、移动、避让、可见。

所以 F5 看不到桃源人是预期结果，不是玩家观察问题。

---

## 2. 本切片目标

本切片不新增大系统，不解封旅人，不做 v3.0-B。

只修两件事：

1. 把 A2 调成封闭生态最小自洽循环，而不是靠外部迁入/迁出。
2. 让鹿拥有繁殖、幼鹿、捕食反馈，而不是“肉更多的羊”。
3. 让桃源人作为“远处有人在看我”真实出现在 live 世界里。

验收句：

> 我能在地图边缘看到三类桃源人，他们保持距离；鹿会吃草、择地、避险、繁殖幼鹿，并被狼/玩家捕猎；羊、兔、鹿、狼和草地资源在封闭桃源地图里能维持一段可解释的动态平衡。

---

## 3. A2.1：封闭生态基线

### 3.1 基本原则

桃花源本轮是封闭生态环境，不把“地图外迁移”作为常规平衡手段。

Cursor 需要先盘点并调试现有生态入口：

- 初始 spawn：羊、兔、狼、鹿、草、树、灌木/浆果等数量。
- 自动迁入：例如羊吸引、狼性别补齐等。
- 自动离开：例如羊无草时离开地图。
- 资源恢复：草、树枝、浆果、蘑菇等是否能恢复。
- 捕食压力：狼对兔、羊、鹿、幼体的优先级。

本切片允许保留少量“入口/出口”代码作为未来地图边界功能，但 live 封闭生态 trace 中不能依赖它们通过。

### 3.2 初始生态调参

先不要盲目加新动物。Cursor 应通过 trace 调整这些初始量：

- 草 / 可食植物数量。
- 羊数量与公母比例。
- 兔数量。
- 狼数量与公母比例。
- 鹿数量与公母比例。
- 水源 / 河岸 / 草地 / 林缘附近资源分布。

建议第一版目标：

```text
草地资源：能支撑羊 + 兔 + 鹿前几分钟消耗，并能恢复
羊：一公一母，允许繁殖羊羔
兔：少量，作为小型猎物和草地压力
狼：一公一母，但捕猎受肉储备/冷却限制
鹿：一公一母，允许繁殖幼鹿
```

具体数值由 Cursor 通过 trace 调，不由本文写死。

### 3.3 封闭生态 trace

新增或补强专项 trace，建议运行至少 5~10 分钟游戏内等价时间或现有可接受的墙钟上限，输出：

- tick / game_day
- grass_count / forage_count
- sheep_count / lamb_count
- rabbit_count
- deer_count / deerFawn_count
- wolf_count / wolfCub_count
- corpse_count / raw_meat_count / cookmeat_count
- hunt_events
- reproduction_events
- starvation_or_exit_events

验收重点：

- 不依赖外部迁入就能维持初始生态。
- 草不应长期归零。
- 食草动物不应快速灭绝。
- 狼不应在肉足够时继续连杀。
- 尸体/肉不应爆炸。
- 鹿至少发生一次繁殖或满足繁殖条件被 trace 明确说明。

## 4. A2.2：鹿的生态循环修正

### 4.1 设计定位

鹿 = 河谷大型野生食草动物。

本项目进入增量阶段后，新增卡必须是机制增量，不是换皮分型。鹿如果只提供更高肉量，就是失败实现。鹿至少要在生态里承担以下作用：

- 消耗草地 / 河岸资源，和羊、兔共享生态压力。
- 依赖安全区域与食物充足繁殖，产生幼鹿。
- 幼鹿跟随成年鹿，是狼更容易选择的猎物之一。
- 成鹿更怕人、火、狼，逃向草地深处 / 林缘 / 河岸。
- 鹿不归属玩家，不进营地，不被玩家“饲养”。
- 鹿的存在让狼群、玩家狩猎、草地恢复、桃源人观察都获得可解释反馈。

最低可接受循环：

```text
草地 / 河岸资源
→ 成鹿觅食恢复体力
→ 安全且食物足够时繁殖幼鹿
→ 幼鹿跟随成鹿成长
→ 狼/玩家捕猎成鹿或幼鹿
→ 鹿尸 / 鹿肉进入食物链
→ 草地压力下降或恢复
```

### 4.2 最小实现

Cursor 按现有结构小改：

1. 新增一张幼鹿卡：

```text
deerFawn / 幼鹿
```

建议标签：

```text
being, animal, herbivore, wildPrey, juvenile
```

建议能力：

```text
capability.move
capability.follow
capability.grow
capability.be_cared_for
capability.be_hunted
```

`deer` 需要补 `capability.reproduce`。若当前 `CardRuleAudit` 缺维度或能力登记，同步补齐。

2. 新增或集中查询：

```gdscript
WorldRules.preferred_terrain_for_actor(actor)
WorldRules.avoid_domains_for_actor(actor)
WorldRules.is_wild_large_prey(card)
WorldRules.is_deer_fawn(card)
WorldRules.reproducing_deer()
WorldRules.deer_fawns()
WorldRules.count_deer_population()
WorldRules.deer_reproduction_allowed()
WorldRules.best_wild_escape_cell(actor, threat)
```

如已有等价函数，可复用，不强求命名。

3. `deer_behavior.gd` 调整：

- 找草时优先选择位于 `terrain.grassland` / `terrain.riverbank` / `terrain.forest_edge` 的草。
- 无草时不要随机乱走，优先向上述 terrain 移动。
- 遇到玩家邻近、狼、火域时，逃向 preferred terrain，而不是只做一步反向移动。
- 鹿不要进入 `domain.camp` 中心，也不要靠近 `terrain.taoyuan_edge`。
- 幼鹿应跟随最近成年鹿；无成年鹿时逃向草地 / 林缘，不原地发呆。

4. 鹿繁殖：

- 初始 2 头鹿必须有性别，至少一公一母。
- 成年公母鹿在以下条件下可产生 `deerFawn`：
  - 附近草 / 河岸资源足够。
  - 附近无狼。
  - 不在营地 / 火域。
  - 当前鹿总数低于上限。
- 幼鹿成长为成年鹿，随机公母。
- 第一版只允许同时存在少量幼鹿，避免指数爆炸。

5. 鹿数量控制：

- 初始 2 头可保留，但必须是一公一母。
- 鹿总量上限第一版建议 4~5。
- 幼鹿上限第一版建议 1。
- 不新增鹿群首领、迁徙季节、复杂怀孕系统。

### 4.3 鹿 trace

补强鹿专项 trace，不只测卡定义：

- 初始世界里应存在鹿。
- 鹿在 300~600 tick 内至少一次进入 `找草` / `吃草` / `逃离狼` / `躲开玩家` / `惧火` 之一。
- 鹿选择的目标草或目标移动方向应优先落在草地 / 河岸 / 林缘。
- 玩家靠近或狼靠近时，鹿距离威胁不应继续缩短。
- 条件满足时能产生 `deerFawn`。
- `deerFawn` 能跟随成年鹿，并能成长为成年鹿。
- 鹿数量不会超过生态上限。

## 5. A3.1：桃源人 live 可见修正

### 5.1 设计定位

桃源人不是测试 fixture，不是只在审计表里存在。A3 的 F5 验收必须能看到：

- 地图远端 / 桃源边界附近有三类桃源人。
- 他们不会进玩家营地。
- 玩家靠近时保持距离。
- 三类人距离差异存在。

### 5.2 最小实现

1. 在 live 初始世界生成三张桃源人卡。

建议位置：

- 靠近 `terrain.taoyuan_edge`
- 不在玩家初始可见核心区
- 不与河流不可站立格冲突
- 能通过相机移动或小地图找到

示例逻辑不要硬编码死坐标到任务层。可在 `world_manager` 初始化阶段通过 `WorldRules.is_taoyuan_boundary_cell(cell)` 寻找合适空格，再 spawn：

```gdscript
taoyuanElder
taoyuanForager
taoyuanYouth
```

2. 桃源人行为：

- 继续只远观 / 避让 / bubble。
- 不交易。
- 不帮工。
- 不接任务。
- 不进入 `domain.camp`。

3. bubble 第一版可以简单，但不能完全空实现。

如果当前 UI 没有通用 bubble 组件，则至少把 `state` / `goal` 写成可见中文短句，例如：

- 长者：`远观篝火` / `保持边界`
- 采集者：`观察取水` / `看着林缘`
- 青年：`好奇张望` / `想再靠近`

### 5.3 social trace 修正

当前 social trace 只检查定义，不够。

必须新增 live 验证：

- `GameState.cards` 里实际存在三种桃源人。
- 三种桃源人的出生点在或靠近 `terrain.taoyuan_edge`。
- 三种桃源人与玩家初始距离满足：
  - elder >= forager >= youth 的距离倾向，或至少各自大于 `actor_social_distance_min`。
- 将玩家移动到其附近后，运行若干 tick，桃源人会进入避让状态或距离不缩小。
- trace 输出明确列出三张卡的位置、距离、状态。

## 7. 验收

### 7.1 自动验收

- L0 PASS，断言数不得回退。
- L1.5：
  - `deer,deerFawn,deerCorpse,deerMeat`
  - `taoyuanElder,taoyuanForager,taoyuanYouth`
- deer trace PASS。
- closed ecology trace PASS。
- social trace PASS，且必须验证 live spawn，不只验证 CardDB。
- L2b live 1×：蘑菇棚仍建成，`USELESS=0`，不复现 shelter 签名。

### 7.2 F5 体感验收

- 玩家能通过移动视野看到桃源三人。
- 桃源三人不进入营地。
- 靠近他们时会退让或保持边界。
- 鹿行为能看出野生大型猎物倾向：远离人 / 火 / 狼，活动更偏草地河岸林缘。
- 鹿能在安全和食物足够时产生幼鹿，幼鹿跟随并成长；狼/玩家捕猎会反馈到鹿数量。
- 生态不是靠不断从地图外刷动物维持；地图里的草、羊、兔、鹿、狼能形成可观察波动。
