# 增量迭代工作流 v0.1

> 目的：把“设计判断 → 工程执行 → 复盘评估”从聊天临时传话，固定成可被 Cursor / Codex / 人工共同读取的文件工作流。

## 1. 当前阶段

项目已进入正式增量迭代。后续增量围绕两条长期主轴：

1. **完整自洽的世界**：更大的地图、更完整的资源基础、生态循环、加工链、前端销售与后端生产基础。
2. **完整的桃源人部落**：桃源人有文化、亲族、资源观、交换观、传统边界与观念变化；玩家卡通过商品、服务、组织方式逐步改变他们。

每次增量可以落在三类内容上：

- **规模**：地图尺寸、区域容量、生态范围、群体数量。
- **新卡**：资源、人物、结构、商品、工具、状态物。
- **新规则**：世界规则、标签、AI 行为、社会反应、生态反馈。

新增卡必须是**机制增量**，不是换皮分型。每张新增卡都要回答：

- 它改变了哪个循环？
- 它消耗或产出什么？
- 它和哪些主体形成反馈？
- 它在世界中如何自我维持、衰减、繁殖、腐坏、迁移或被转化？
- 如果把它删掉，游戏机制是否少了一种判断？

如果答案只是“肉更多 / 颜色不同 / 另一种同类目标”，则不应作为正式增量通过。

## 2. 固定工作流

每一轮按以下顺序推进：

```text
Codex 写单次增量方案
↓
用户确认
↓
Codex 写入本地文档
↓
Cursor 按文档实现
↓
用户让 Codex 读取实现结果
↓
Codex 复盘：目标达成度、风险、下一步
```

用户不需要在 Codex 和 Cursor 之间复制大段内容。Cursor 直接读取本地文档。

## 3. 单次增量方案必须包含

每次方案至少写清：

| 项 | 内容 |
|----|------|
| 目标 | 本轮为什么做，服务哪条长期主轴 |
| 范围 | 本轮只做什么，不做什么 |
| 类型 | 规模 / 新卡 / 新规则，可能多选 |
| 卡牌 | 新增或修改哪些卡，预期 `factory_lane`、标签、能力 |
| 世界规则 | 需要新增或修改哪些 `WorldRules` 查询 |
| 机制增量 | 新卡改变的循环、约束或反馈；避免换皮分型 |
| AI 可达性 | 玩家任务、生态行为、环境自动、手动交互分别如何触发 |
| Cursor 执行指导 | 改哪些文件，避免哪些做法 |
| 验收 | L0 / L1.5 / L2b / 专项 trace / F5 体感 |
| 风险 | 哪些做法会造成长期耦合或未来卡死 |

## 4. Cursor 执行纪律

Cursor 实现时必须遵守：

- 新卡先走 `docs/design/card-rule-audit.md` checklist。
- 新卡必须具备机制增量说明；生态卡尤其必须说明繁殖 / 死亡 / 消耗 / 捕食 / 迁移 / 恢复中的至少一个闭环位置。
- 可用 `tools/run_card_factory_scaffold.ps1` 生成脚手架，但脚手架不替代玩法设计。
- 新卡 / 卡牌规则切片至少跑 L0 + L1.5。
- 改 need / craft / 主链 / 生态行为时，必须补对应 L0，并跑 L2b 或专项 trace。
- 不在 `scripts/player`、`scripts/world`、`scripts/ui`、`scripts/input` 新增散落 `card_type ==` 判断；需要新增规则查询时优先进入 `WorldRules`。
- 不一次性做“大地图 + 50 人 + 完整社会 + 新商品 + 新 AI”。
- 遇到规则不清时停止并写明阻塞，不用临时硬编码绕过去。

## 5. 当前第一切片

第一轮正式增量切片已定为：

```text
v3.0-A：四倍河谷生态 + 桃源三性格远观
```

工程上不得一次合入完整 v3.0-A。必须拆成：

```text
A1 terrain / 4 倍地图
A2 鹿生态链
A3 桃源远观 / 旅人封存 / 桌子 dining
```

具体策划案与 Cursor 执行单：

```text
E:\桌面\方寸商国：桃花源记\docs\design\v3.0-a-river-valley-ecology-taoyuan-archetypes.md
```

定位：

- 主目标：把地图从测试棋盘推进到 4 倍面积河谷，建立空格天然属性、资源带和生态分布。
- 副目标：新增三类桃源人远观原型，先表现边界与态度，不做员工 / 交易 / 部落系统。

原则：

- 先 4 倍面积河谷，不做 4–6 倍完整世界。
- 先鹿这一条完整生态位，不同时扩狐狸、鱼、野猪等多条动物线。
- 先桃源三性格远观 / 避让 / 评论，不做 50 人社会与完整观念系统。
- 旅人封存；桌子先作为 `service.dining`，不承担商业主舞台。
- 先证明“世界事实 + 社会反应”可以小切片稳定接入。
