# 《方寸商国》意向系统与世界演化设计总结

> 策划向主文档（v0.3）。**正本：** `docs/design/core/意向系统与世界演化_v0.3.md`

---

## 1. 核心定位

《方寸商国》不是传统任务驱动游戏，不应该做成：

```text
NPC 发布任务 → 玩家收集材料 → 条件满足 → 任务完成 → 发奖励
```

而应该做成：

```text
小世界自运行
↓
不同主体产生需求与意向
↓
意向之间发生冲突、合作、交易、误解
↓
玩家扮演「系统」改变世界条件（奖励式上帝视角）
↓
玩家卡基于自身判断形成行动承诺
↓
世界关系发生变化
↓
新的意向和后果继续涌现
```

核心词是 **意向 Intent**，不是 Quest。  
但 **需求 Need 与意向 Intent 不冲突**——见 §1.1。

---

## 1.1 需求与意向：基本面与升华面

```text
需求（Need）= 基本面
  例：人物卡需要吃肉 → 饥饿、营养、安全

意向（Intent）= 需求的升华面
  例：如何成规模、产业化地满足吃肉
       → 养殖 → 屠宰 → 餐饮 → 岗位 → 合约 → 信用
```

关系：

| 层级 | 回答的问题 | 例子 |
|------|-----------|------|
| 需求 | 缺什么？不做会怎样？ | 饿了、冷了、有狼 |
| 意向 | 值不值得做？怎么做才划算？和谁有关？ | 建蘑菇棚稳食、雇人守棚、用木头换劳动 |

**实现原则：**

- 底层可以仍是「需求扫描 + 优先级」——这是正确的第一步。
- 上层逐步加上「价值判断、关系阻力、产业设想、后果预期」——这才是意向。
- 玩家看到的 bubble / goal，应优先表达 **意向层**；需求层只在紧急时占主导（饥寒、遇狼）。

```text
不可见 → 已感知 → 可理解 → 可设想 → 可行动 → 可产业化
         ↑ 需求多在左段    ↑ 意向多在右段
```

## 2. 世界观与地图阶段

玩家卡：躲避战乱逃入桃花源的南宋商人，懂经营、组织、工艺、交易。

桃源：**不是空地图**，而是已有秩序的小社会（目标规模约 **50 人**）。  
他们有亲族、资源观、交换观、地方知识、传统价值；不是员工池，也不是低智 NPC。

**地图阶段规划：**

- 当前地图 → 扩大 **4～6 倍**，以容纳河流、山林、草地、山岩、园圃与聚落。
- **蘑菇农**是桃源人中的一员，不是外来系统单位；后续桃源人秩序与生态同步丰富。
- 玩家商人 **初来乍到**：桃源人对默认态度为 **谨慎观察、有明确边界感**——不是封闭拒绝，也不是无脑帮助。

玩家卡逐渐把商业逻辑、生产组织、货物流通带入桃源；方向是：

```text
自足小世界
↓
与桃源人初接触、互惠（约 1 游戏小时内玩家应感到的爽点）
↓
建立生产与岗位
↓
物物交换 → 契约 → 信用 → 婚姻亲族
↓
旅人由少到多，角色由「误入者」到「观光者」到「商机与冲突载体」
↓
约 10～15 游戏年：巨型商业都市，阡陌交通，千行百业
```

**无通关**：本作是 **故事生成器**，可一直经营；在达到终极形态（巨型商业体）之前，有 **阶段性目标**，由意向驱动，而非任务列表。

## 3. 玩家与玩家卡的关系（策划共识）

```text
玩家 = 玩家卡本人（第一人称体验）
上帝视角 = 系统对玩家卡的「奖励能力」
```

- 拖拽、点选、放置等 **上帝视角操作**，在叙事上 **全部归因于玩家卡**——玩家卡感觉不到「头顶还有个玩家在操控」。
- 玩家实际在扮演某种 **「系统」**：改布局、改条件、改机会；玩家卡则在世界里 **思考、承诺、执行、承担后果**。
- UI 气泡、goal、内心独白，一律用 **玩家卡口吻**（「我想……」「也许可以……」），不用「系统提示玩家」口吻。

---

## 4. 核心设计原则

### 4.1 不做任务系统，做意向系统

见 §1.1。意向是主体基于世界条件、知识、价值观、关系、风险和机会产生的 **行动倾向**。

示例（复合弓，中后期内容）：

```text
玩家卡发现：有木、有火、有牛筋、懂弓结构、猎手阿牛想要更强工具
→ 意向：「也许能试做一张更强的弓；若阿牛愿以劳动交换……」
```

### 4.2 世界条件成熟后，才产生产业级意向

有盐地但无陶锅、无燃料、无腌肉需求 → 最多「这里的土有咸味」，不直接刷「发展盐业」意向。

### 4.3 发展是阴阳，不是纯正向

玩家为事业触犯桃源传统时：

- **若玩家最终达成目标**，且 **桃源整体需求大于反对**（支持 + 妥协 > 反对）→ 反对者可能付出 **血的代价**。
- **若未达成或代价过大**，则出现分裂、妥协、补偿、仪式、补种等 **关系修复或永久裂痕**。

**「血的代价」的边界（策划共识）：**

- 后果来自 **人物卡内在逻辑**——性格、观念、认知、立场、关系网共同推出来的 **命运感**，不是作者强行写死谁必须死。
- 目标不是「在电脑里做一个真人」，而是 **感受上像人**：玩家能读出「他为什么反对、为什么遭殃、为什么有人支持」。
- 当前人物卡仍是 **高级生物**（需求、行动、关系），**观念层尚未成型**；后续逐步加入：价值倾向、对传统/新秩序的态度、对商人或具体政策的立场。
- 极端后果（受伤、死亡、被排挤、亲族决裂）是 **逻辑链末端的可能结果**，不是默认每局必演的血腥开关；同一冲突在不同人物上可以是社会性代价或物理性代价。

几乎所有资源 **都可以利用**；差别在于 **利用方式、社会代价、生态后果、接受程度**——用 **关系阻力**，不用大量硬性禁忌。

### 4.4 第一个全游戏意向样板（当前优先级）

**首选：自给自足的生存与安全**（火、 shelter、工具、稳定食物链）。  
在此之上才是：蘑菇棚等稳定食物 → 商品（熟肉/工具）→ 与桃源人的劳动交换。

## 5. 玩家卡的判断逻辑（商人价值观）

除饥饿、危险外，还应体现：

```text
生存价值 / 稳定价值 / 剩余价值 / 商品价值 / 信用价值
组织价值 / 基础设施价值
关系风险 / 生态风险 / 外部风险
```

玩家卡应像商人：**会算账、会组织、会看人、会把资源→商品→信用→关系→岗位→生产体系**。

---

## 6. 桃源人的判断逻辑

桃源人聪明方向 **不同于** 玩家：懂本地生态、河流、亲族、习俗、地方秩序。

- 默认对商人：**谨慎观察，边界清晰**。
- 约 **50 人** 小社会：有长者、猎户、农人、手艺人、年轻人等 **群体倾向**，不必先做单一大 NPC；「第一个会拒绝你的人」来自 **社会结构**，不是单个任务发布者。
- 蘑菇农是其中一员；雇佣、木头契约、守棚等，是 **社会合约的雏形**。

### 6.1 第一批愿意帮玩家事业的人，靠什么？

**核心理由：在玩家卡这里得到了「自然界 + 旧桃源秩序」里不存在的东西。**

| 类型 | 例子（随阶段解锁） |
|------|-------------------|
| **商品** | 稳定熟肉、盐渍物、更好工具、统一规格的布/陶 |
| **服务** | 托管、兑换、储存、代办、护运、计时交货 |
| **交互方式** | 契约、信用、岗位、分工、可预期的交换规则 |

不是「发布任务帮商人」，而是：**旧生活解决不了的问题，在新秩序里第一次被解决**——帮助者往往也是第一批 **受益者 + 见证者**。

**1 小时爽点设计含义：** 玩家应体验到至少一次「我带来的东西，让某个桃源人愿意靠近、试用、再帮一把」——而不是空喊「好感度 +10」。

## 7. 世界母系统（地图扩大后逐步落地）

```text
1. 河流水文    2. 森林木材    3. 草地畜牧
4. 山岩矿物    5. 农业园圃    6. 人群与文化
```

三流：

```text
物流：材料 → 加工 → 商品 → 仓储 → 销售
人流：桃源人 / 岗位者 / 旅人 / 商队 / 家庭
生态流：河流 → 草木 → 动物 → 食物 → 风险
```

前端服务与后端生产见原架构（桌子、馆店、渡口 / 蘑菇棚、牧场、陶坊……）。

---

## 8. 旅人与商队的时间线

**世界铁律：桃源人不会让任何一个人走出桃花源。** 外来者一旦误入，即 **困于小世界**；「离开」不是早期选项。

| 阶段 | 游戏进程（大致） | 旅人/外来者角色 |
|------|------------------|-----------------|
| 早期 | 生存与初接触 | 极少；**迷路者**。可能：抱团、向玩家卡求援、死于自然系统、被人杀死。玩家商业体系尚不完善，**不是**早期主舞台 |
| 中期 | 商业初建、思维转变 | 类似 **观光/过客**；桃源内部因新商品与服务产生 **观念松动**——旧秩序外的「优异生活样本」进入日常 |
| 后期 | 产业发达、枢纽形成 | 商机、新冲突、可 **丰富/改造世界** 的要素；外部世界以 **向内带来的机会与风险** 形式存在，而非物理出境 |

**观念变迁路径（桃源人侧）：**

```text
旧秩序 + 自然供给
↓
玩家卡带来新商品 / 新服务 / 新交互方式
↓
部分人先得到稳定满足与新的生活快感
↓
观念从谨慎、边界感 → 向往「外部向内」带来的机会
↓
支持 / 妥协 / 反对 重新分野（阴阳代价在此发生）
```

早期旅人 **不是** 主导乐趣；**1 小时爽点** 在 **桃源人是否因「从未有过的东西」而愿帮玩家事业**。  
对迷路者：不预设「玩家必奴役」；其命运由 **自然、抱团、冲突、人物内在逻辑** 涌现——可以是求援、被利用、被排斥或死亡，由世界与角色状态决定。

## 9. 技术架构方向（策划可读版）

```text
世界事实 WorldModel
↓
知识/认知 KnowledgeModel（per 主体，非全知）
↓
需求扫描 + 机会扫描
↓
意向生成 IntentEngine
↓
商人/桃源人价值判断
↓
气泡表达 SpeechBubbleBrain
↓
承诺 Commitment
↓
TaskFSM 具体执行（沿用左右键语法）
↓
后果 ConsequenceLedger → 其他主体重新评估
```

**Intent 管「为什么做」；TaskFSM 管「怎么做」。**  
底层执行：**左键 = 物理作用，右键 = 关系作用**；AI 与玩家共用，不能凭空生成结果。

与现有工程关系：**需求层 ≈ 现有 need 系统；意向层在其上升华；TaskFSM ≈ 现有 craft_task**——不另起炉灶，逐步改名与加评分/后果。

## 10. 数据结构（摘要）

- **CardDef**：卡牌标签与层级（wood、fire、tool……）
- **KnowledgeDef**：某主体 **是否知道** 某配方/工艺（非全局 unlock）
- **IntentProposal**：id、domain、required_facts/knowledge、score、reasons、risks、blockers
- **ContractProposal**：双方 gives、trust、duration、risks（劳动换工具、物资换守夜等）
- **ConsequenceLedger**：行动后果摘要，供后续意向重算（需摘要+衰减，非无限 log）

---

## 11. SpeechBubble 原则

来自 **当前最高分意向** 或 **阻塞原因**；须符合 **该主体已知事实**（未感知的不冒泡）。

例：

```text
「有木、有筋、有火……也许能试做一张更强的弓。」
「这里像有盐，但现在还缺取卤和煮盐的法子。」
「熟肉能卖，但肉源还不稳。」
「蘑菇棚能养人，但木头消耗会变大。」
```

---

## 12. 故事机逻辑

```text
世界条件成熟 → 主体产生需求/意向 → 行动 → 关系改变
→ 支持/反对/妥协 → 后果（含阴阳代价）→ 新意向 → 循环
```

无通关；**10～15 游戏年** 可达终极形态：**巨型商业都市，各司其职**。此前每阶段有 **意图驱动的里程碑**（稳食、稳商、通商、枢纽……），非任务板勾选。

---

## 13. 最小实现路线（修订版）

与 §4.4 对齐，顺序如下：

### Phase 0：生存与安全（**工程基线已完成，策划 Layer 1 待推**）

```text
火 /  shelter / 基础工具 / 防狼 / 稳定食物感知
```

对应现有：篝火、草棚、刀矛、蘑菇棚链等。**需求层与 survival 主链已在 v2.70 跑稳**（L0 625 / L2b 3× PASS）。  
策划下一重心：Layer 1 v3.0「第一个游戏日稳稳活下来」的感受验收，或并行启动 Step 1 意向 Debug 面板。

### Step 1：Debug 意向面板（不改玩法）

在现有 need 之上 **显示** 候选意向、blockers、score——便于策划验收「像不像在琢磨」。

### Step 2：SpeechBubble 模板化

goal / 气泡从意向 reasons 生成，统一玩家卡口吻。

### Step 3：三个意向模块（生存稳态之后）

```text
1. 稳定食物（蘑菇棚及维护）
2. 商品（熟肉 / 工具）
3. 合作（桃源人以劳动换物资；蘑菇农为首个社会样本）
```

### Step 4：Commitment → TaskFSM

承诺后才拆步骤（找木、制斧、砍树、雇人……）。

### Step 5：ConsequenceLedger

例：蘑菇棚建立 → +稳定食物、+桃源人好奇；-木材压力、-野采价值。

### Phase Map：4～6 倍地图 + ~50 桃源人雏形

与 Step 3～5 **并行规划**，内容逐步接入；蘑菇农归入桃源社会，非孤立系统。

---

## 14. 工程原则（给实现时的边界）

不要写成单一 if-else 巨兽；应：

```text
世界事实 → 模块生成候选 → 统一评分 → 承诺 → TaskFSM → 后果 → 重评估
```

**策划验收问句：**

1. 玩家卡是在 **应急**，还是在 **琢磨一门生意**？
2. 桃源人是在 **被动配合**，还是在 **有立场地支持/拒绝/要价**？
3. 玩家成事后，是否 **有人付出可见代价**（阴阳）？
4. 1 小时时，是否至少一次 **「有桃源人愿意帮我的事业」**？

---

## 15. 最终目标（一句话）

**《方寸商国》不是任务驱动游戏，而是一个可长期经营的故事生成器：小世界里的多个主体，根据各自需求、认知、价值观与关系压力，不断产生意向、行动、交易、冲突与后果——玩家就是玩家卡，在约 10～15 年的尺度上，把桃源从自足小世界带向千行百业的商业枢纽，且发展永远伴随代价。**

## 16. 工程基线与增量迭代（2026-05-27，v2.70）

> 本节给 Codex / Cursor 接续用；策划向 §13 路线不变，实现方式已切换为「小切片 + 规则枢纽」。

### 16.1 当前架构

```text
NeedEvaluator → MissionResolver → ActionRunner + ExecutionStack
  → CraftTask FSM → CampHelpers / Interaction
规则查询枢纽：WorldRules（facade）+ 7 子模块（craft / service / state / material / camp / commerce / ui）
```

### 16.2 阶段切换

| 阶段 | 内容 | 状态 |
|------|------|------|
| v2.x 耦合收口 | runtime 决策层去 `card_type` 硬编码；寻路 / UI / 营地域 / 经营规则化 | **已完成** |
| **增量迭代** | 新卡 / 新玩法：checklist → WorldRules API → L0 + L2b | **当前** |
| v3.x 策划层 | 稳态 / 第一商品 / 可见意向 / 小社会 | 待排期 |

### 16.3 增量迭代守则（摘要）

1. 新卡先填 `docs/design/card-rule-audit.md` checklist  
2. 规则只增 `WorldRules.*`；禁止在 player/world/ui/input 新增 `card_type ==`  
3. 改 need 必同步 `need_contract.gd` + L0 契约测试  
4. 每切片：CHANGELOG + `CODEX_HANDOFF.md` §12  

### 16.4 验收基线

- L0 unit：**625** assertions PASS
