# 《方寸商国》分层构建路线图 v3.x

> 策划向实施计划。与《意向系统与世界演化 v0.3》配套。  
> **正本：** `docs/design/core/构建路线图_v3.x.md` · 入口 `docs/README.md`  
> 原则：**一口吃不成胖子，一层只回答一个问题。**

---

## 总原则

每一层都要满足：

1. **可玩闭环** — 玩家能玩到一个完整小循环  
2. **只加 1～2 个「旧桃源没有的东西」**  
3. **验收用感受** — 不堆功能清单  
4. **可停手** — 本层完成即可发布/试玩，不依赖下层  

**四层不做：**

- 不做完整 50 人社会再开玩  
- 不做完整意向引擎再开玩  
- 不做 4～6 倍地图一次铺完  
- 不做复合弓 / 盐业 / 娱乐 / 枢纽同期上线  

**三条并行线（每版只主推一条）：**

```text
A. 世界事实线   地图 → 资源带 → 生态
B. 社会线       人数 → 群体 → 观念 → 后果
C. 商人线       生存 → 第一商品(腊肉) → 交换 → 岗位 → 产业
```

## Layer 0：v2.x 收尾（**已完成 → 增量迭代阶段**）

### 目标

把已有 MVP 当地基打实，不再叠新系统。

### 范围

- 生存链：火 / 草棚 / 刀矛 / 蘑菇棚 / 营地储存 / 基础 AI  
- 修复：任务抢占、伐木制斧、狼窝、建造选址等「会晃、不执行」类问题  
- **工程收口（v2.64–v2.70）**：`WorldRules` 7 子模块 facade、寻路规则化、UI/拖拽/badge 规则化、`CardRuleAudit` 守卫  

### 验收

- [x] 稳定玩若干游戏日，逻辑不崩  
- [x] 能走到「想建蘑菇棚 / 想卖熟肉」而不卡死  
- [x] headless L0（625 assertions）+ L2b live 可 PASS  

### 本层不做

腊肉、50 人、意向面板、地图扩大  

### 状态

**已完成（2026-05-27，v2.70）** — 耦合风险方案内 100%；L0 625 / L2b 连续 3× PASS。  
**当前阶段：增量迭代** — 新功能走 `card-rule-audit.md` checklist + WorldRules 小切片 + L0/L2b；**下一主推 Layer 1 v3.0 稳态或 Layer 3 v3.2 可见意向**（策划排期，见下）。

## Layer 1：v3.0「稳态」— Phase A 生存与安全

### 要回答的问题

玩家卡是不是可靠的「高级生物」？**第一个游戏日能否稳稳活下来？**

### 策划内容

| 要素 | 定义 |
|------|------|
| 火 | 有稳定热源、防狼、可烹饪 |
| shelter | 草棚 / 营地概念成立 |
| 工具 | 刀 + 矛（或等价猎具） |
| 安全 | 遇狼有应对，不必每夜赌博 |
| 食物 | **感知**到稳定食物来源（不必已是产业） |

**熟肉 = 活命**，不是商品符号。  
**桃源人 = 远观 + 谨慎**，本层不展开社会戏。

### 需求优先级（策划规则）

```text
饥 / 寒 / 狼 / 立即危险  >  建棚 / 工具  >  建桌 / 蘑菇 / 经营
```

### 玩家卡表达

- goal / 气泡统一 **第一人称**（「我想…」「也许可以…」）  
- 上帝视角操作 **归因于玩家卡**  

### 验收句

> 「我不用微操，他也像个会在桃源活下来的人。」

### 本层不做

- 腊肉量产  
- 桃源人帮工、合约  
- 观念、阴阳代价  
- 地图扩大  

### 主推并行线

**C 商人线（生存）为主**；A/B 只维护不扩展

## Layer 2：v3.1「第一商品」— 腊肉（1 小时爽点）

### 要回答的问题

第一件 **旧自然 + 旧秩序没有** 的东西是什么？

### 定锚：腊肉（条件组合型储存肉）

- 桃源 **有条件**（肉、火、烟、盐味、时间…）  
- 需玩家卡 **产品化**（工序、容器、可重复）  
- **≠** 人人会做的熟肉  

### 三层 unveil

1. **好奇** — 第一块「能存住、味道不同」的肉  
2. **试用** — 断粮 / 待客 / 办事中真正用到  
3. **靠近** — 劳动 / 原料换稳定供应或工序  

### 第一批接触者（2～3 类即可）

- 需储粮、待客者  
- 闻味好奇的年轻人  
- cautious 长者（不帮但认可）  

### 验收句（1 小时爽点）

> 「我做出了熟悉环境里长不出来的东西，有人愿意靠近、试用、再帮一点。」

### 本层不做

复合弓、娱乐、审美、完整盐业、旅人主舞台、地图 4～6 倍  

---

## Layer 3：v3.2「可见的意向」

### 目标

几乎不改玩法，改 **可读性** — 人物像在「琢磨」。

### 内容

- bubble：为什么想做 / 为什么卡住  
- 可选 Debug：候选意向列表（策划验收）  
- 区分 **需求句** vs **意向句**  

### 验收句

> 「他停住时，我能看懂是在等盐、等肉，还是在等谁点头。」

### 依赖

v3.1 已有具体商品，意向才有内容  

---

## Layer 4：v3.3「小社会」

### 4a 地图

- 先 **2 倍** 可玩聚落 + 资源带  
- 再扩向 4～6 倍（河/林/草/山）  

### 4b 人

- 先 **8～12 人**，再向 50  
- 蘑菇农 = 桃源人一员  
- 群体：长者 / 猎户 / 主妇 / 年轻人 / 手艺人  

### 验收句

> 「这是有小秩序的地方，不是开荒模拟。」

---

## Layer 5：v3.4「新交互」

- **桌子** = 公开、可重复、可预期交换  
- **岗位** = 守棚、送柴、帮熏（蘑菇农为模板）  
- **轻契约** = 给 X 做 Y 做 Z 天  

### 验收句

> 「帮忙是讲规矩的互换。」

---

## Layer 6：v3.5「阴阳后果」

- 轻量观念标签（2～3 个/人）  
- 支持 / 反对 / 妥协  
- 后果：排挤、断供、决裂 → 极端由逻辑链推出  

### 验收句

> 「有人因新秩序得利，有人因立场遭殃——读得懂为什么。」

## 需求 → 意向 → 商品（全路线参考）

```text
Phase A  生存与安全     火、棚、工具、防狼
Phase B  稳定与储存     腊肉、盐渍、桶、仓
Phase C  交换与规则     桌子、契约、岗位
Phase D  生活与审美     染布、器型、节庆
Phase E  闲暇与娱乐     茶、酒、棋、说书
Phase F  权力与象征     复合弓、礼器、枢纽
```
