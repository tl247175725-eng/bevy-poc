《方寸商国：世界规则圣经 v0.3》

# 1. 项目核心

《方寸商国》核心不是生存。

核心是：

- 关系
- 状态跃迁
- 环境
- 需求
- 涌现

玩家经营的是：

“一个会自然循环并产生经济行为的生态世界”

# 2. 世界层级

## 2.1 地形层

不可轻易搬运的世界基础：

- 荒地
- 池塘
- 山岩
- 迷雾

特点：
- 不属于卡牌
- 不可堆叠
- 提供环境条件

## 2.2 覆盖层

附着于地形上的环境覆盖：

- 草皮
- 泥浆
- 灰烬

特点：
- 可移动
- 可叠加
- 可参与关系

## 2.3 实体层（卡牌）

例如：
- 木头
- 石头
- 工具
- 羊
- 玩家
- 木炭

特点：
- 有碰撞
- 可拖拽
- 可叠加
- 可改变本质

## 2.4 领域层

环境持续影响范围：

- 热
- 湿
- 光
- 安全

例如：

篝火 → 热力领域

# 3. 核心哲学

## 3.1 游戏核心是“关系”

不是资源数字。

而是：

“关系驱动的状态变化”

## 3.2 游戏核心是“本质跃迁”

例如：

木头 + 篝火 → 木炭

不是：

燃料值+10

## 3.3 数量只影响量产能力

数量：
- 不决定是否成立
- 只决定规模

例如：

1木头也能形成草棚

# 4. 左右键语义

## 左键 = 物理作用

代表：
- 砸
- 推
- 攻击
- 强制施力

特点：
- 有方向
- 有阻挡
- 有节奏
- 有碰撞

### 左键砸击规则

所有砸击：

至少2下触发结果

流程：

第一次进入碰撞
→ 红色数字 1
→ 抖动

离开碰撞

第二次进入碰撞
→ 红色数字 2
→ 触发结果

持续接触：
不重复计数

## 右键 = 关系作用

代表：
- 叠加
- 依附
- 状态改变
- 组合

特点：
- 幽灵移动
- 无碰撞

# 5. 右键关系规则

## 5.1 同类关系

同类卡 → 数量叠加

例如：

木头 + 木头 → 木头×2

## 5.2 有关系

不同卡 → 本质变化

## 5.3 无关系

弹开到最近空格

# 6. 右键关系方向规则

默认：
所有关系双向成立。

例如：

木构件 + 木头 → 桌子
木头 + 木构件 → 桌子

三角碎石 + 木头 → 斧头
木头 + 三角碎石 → 斧头

## 单向关系

只有存在明确：
- 容器
- 目标
- 环境

时才单向。

例如：

空桶 + 池塘 → 一桶水

# 7. 移动规则

所有生物：
- 只能上下左右移动
- 禁止斜向移动

包括：
- 玩家
- 羊
- 旅人
- 商人

# 8. 生物寻路规则

所有生物：
目标驱动寻路

不是随机漂浮。

## 羊规则

羊会：

自动检测所有非迷雾中的草皮

行为：

发现草
↓
锁定最近草皮
↓
四方向寻路
↓
绕开障碍
↓
靠近草
↓
进入“吃草”

## 羊状态机

当前：
- 找草
- 吃草
- 眩晕
- 逃跑

## 羊逃跑

被利器攻击：

羊进入逃跑状态
↓
高速移动
↓
逃向迷雾
↓
离开地图

不是瞬移消失。

# 9. 玩家规则

玩家不是UI。

玩家是：
世界中的实体人物卡

因此：
- 必须 physically 到场
- 必须寻路
- 有空间位置

## 玩家移动速度

当前MVP统一：

0.35秒 / 格

# 10. 世界关系语法

世界关系不是配方。

而是：

“属性逻辑”

## 当前确认属性

- 尖锐
- 坚硬
- 钝重
- 木柄
- 热
- 湿
- 结构

# 11. 中介物体系

中介物：
连接基础资源与高级结构

当前：
- 碎石
- 木构件
- 木炭
- 半桶水

# 12. 时间规则

现实1秒 = 游戏1分钟

## 热力蒸发

一桶水
30秒 → 半桶水
60秒 → 空桶

# 13. 旅人系统

旅人不是商人。

旅人是：
观光者 / 潜在消费者

## 旅人行为

闲逛
↓
发现经营单位
↓
被吸引
↓
走向经营单位
↓
提出需求

## 经营单位

只有存在经营单位：

旅人才会进入消费行为

例如：
- 桌子
- 餐区
- 篝火区

## 旅人需求

当前占位：

“有没有吃的？”

需求目标：
熟肉

# 14. 状态可观察原则

所有重要状态：
必须可查看

例如：

羊
状态：眩晕

以后：
- 饥饿
- 害怕
- 舒适
- 工作中

都必须可观察。

《MVP测试目标 v0.3》

# 本轮目标

让玩家能完成一条清楚可验证的商业闭环：

砸石
↓
塑形小刀
↓
击晕羊
↓
屠宰出肉
↓
烤成熟肉
↓
制作桌子
↓
桌子吸引旅人
↓
旅人提出“有没有吃的？”

# 本轮施工规则

## 1. 水蒸发补齐

一桶水处于篝火 3×3 热力领域内：

30秒 → 半桶水
60秒 → 空桶

半桶水继续处于热力领域内：

30秒 → 空桶

水桶离开热力领域：

保留当前蒸发进度，不继续计时。

## 2. 旅人触发修正

旅人发现经营单位的条件：

存在桌子即可。

旅人靠近桌子后提出：

“有没有吃的？”

需求目标：

熟肉

熟肉不是触发旅人的前置条件。

## 3. MVP迷雾替代规则

当前地图没有迷雾层。

羊逃跑临时规则：

逃向地图边缘出口
↓
到达边缘后离开地图

后续版本再替换为“逃向迷雾”。

## 4. 状态可观察补齐

选中卡牌时必须显示：

- 状态
- 位置
- 数量
- 当前目标
- 热力蒸发进度

## 5. 右侧试玩清单

页面右侧必须显示本轮试玩链路：

1. 石头砸石头两下，得到碎石
2. 坚硬物砸碎石两下，选择石制小刀
3. 石头砸羊两下，羊眩晕
4. 小刀砍眩晕羊，得到肉×2
5. 肉右键放到篝火，得到熟肉
6. 木头砸木头两下，得到木构件
7. 木构件与木头右键组合，得到桌子
8. 旅人靠近桌子，提出“有没有吃的？”

# 本轮验收

玩家不看代码，只看页面，也能理解：

- 下一步要试什么
- 当前卡牌是什么状态
- 旅人为什么被吸引
- 水为什么正在变化

《MVP经营单位规则 v0.4》

# 1. 桌子是基础经营单位

桌子不是单纯装饰物。

桌子是当前 MVP 的最小经营单位。

# 2. 经营单位检测范围

经营单位会检测自身旁边最近 1 格。

只要相邻 1 格内存在消耗物，经营单位就可以把它作为候选商品。

当前消耗物：

- 熟肉

# 3. 需求匹配

旅人需求由人物卡自身生成和保存。

当前默认需求：

熟肉

当旅人靠近桌子，且桌子旁边 1 格存在熟肉：

桌子 + 熟肉
↓
被黄色关系线与黄色框标记
↓
旅人自动消费熟肉
↓
黄色关系标记消失
↓
旅人需求清空
↓
旅人进入闲逛状态

除非旅人卡之后生成下一个需求，否则不会继续消费。

# 4. 左右键拖拽确认

左键拖拽：

- 只表示物理碰撞
- 卡牌在松手后停留在拖到的位置
- 结果只由碰撞规则产生
- 如果目标格不可用，则停在最近可用格

右键拖拽：

- 表示关系放置
- 使用幽灵拖拽
- 松手后落到目标格或触发关系
- 不弹回原位

# 5. 重置按钮

右侧界面需要提供“一键重置”按钮。

点击后：

- 清空当前局面
- 恢复初始卡牌
- 清空日志
- 重建旅人默认需求

《MVP生态闭环第一步 v0.6》

# 本轮目标

让斧头和水都进入生态循环。

本轮重点不是新增大量内容，而是让已有链条闭合：

树 → 木头 → 桌子

水 → 草 → 羊 → 肉 → 熟肉 → 旅人

# 1. 树规则

树是卡牌。

但树不是普通可搬运卡牌。

树属于：

- 实体层
- 扎根实体
- 木材来源

规则：

- 树有位置
- 树可被查看
- 树不可拖拽
- 斧头左键砍树 2 下
- 树 → 木头×2
- 斧头保留

# 2. 旅人意志规则

旅人不是玩家资源。

旅人属于自主行为体。

规则：

- 旅人不可拖拽
- 旅人由自身需求和状态机移动
- 玩家只能通过改变环境、商品和经营单位间接吸引旅人

# 3. 浇水生草规则

一桶水可以右键放到空地。

放置后，玩家对一桶水进行左键操作：

第一次：

红色数字 1

第二次：

红色数字 2
↓
该地生成草皮
↓
一桶水变回空桶

这不是配方。

这是：

水对土地发生湿润关系。

# 4. 草与羊规则

草不是无限资源。

羊吃草后：

- 草皮消失
- 羊进入吃饱/闲逛或离开状态

如果地图没有草：

- 羊会离开当前区域

当新草皮出现：

- 可以吸引新的羊到场

# 5. 池塘与雨

完整生态阶段：

池塘应当有水量。

空桶取水会让池塘从：

满 → 浅 → 干

下雨会：

- 恢复池塘
- 生成草皮或湿地

当前 MVP 暂不实现池塘水量。

本轮只实现：

空桶可从池塘取水。

# 6. 本轮验收

玩家应能看出：

- 斧头为什么有用
- 水为什么有用
- 草为什么不是无限的
- 羊和草之间开始形成生态关系

《MVP时间天气与堆叠拆分 v0.7》

# 本轮目标

解决三件事：

- 同类堆叠后可以拆出单张
- 羊吃草进入明确时间系统
- 池塘会被取水消耗，并由下雨恢复

# 1. 堆叠拆分

右键：

同类卡牌叠加为数量。

左键：

当一叠卡牌数量大于 1 时，拖拽会从堆叠中拆出单张。

原堆叠：

数量 -1

新卡牌：

数量 = 1

这代表玩家从一堆物品中拿出一个实体单位。

# 2. 时间比例

当前 MVP 时间比例：

现实 1 秒 = 游戏 1 分钟

现实 24 分钟 = 游戏 24 小时

这个比例用于让 24 小时制有意义，同时保留玩家可操作的体感时间。

# 3. 羊吃草时间

羊吃完 1 块草皮需要：

现实 1.5 分钟

对应游戏 1.5 小时。

这个时间是当前生态消耗循环的基准。

后续类似消耗行为，都应优先参考“羊吃草皮”的体感长度。

吃完后：

草皮消失。

# 4. 池塘水量

当前 MVP 池塘容量：

3 桶水

空桶取水 1 次：

池塘水量 -1

池塘状态：

- 满：3
- 浅：1-2
- 干：0

池塘干涸时：

空桶不能取水。

# 5. 天气

当前 MVP 天气：

晴天 / 下雨

天气按游戏时间循环。

暂定：

每 6 游戏小时触发一场雨。

下雨会：

- 补满池塘
- 随机生成 3 块草皮
- 在日志中提示

后续可扩展：

- 下雨自然生成草皮
- 下雨形成湿地/泥浆
- 雨量影响游客行为

# 6. 一桶水规则

一桶水 = 一块草皮。

一桶水放在空地后，双击使用：

生成 1 块草皮
↓
水桶变空

《MVP生态吸引与旅人交易 v0.8》

# 本轮目标

让生态和经济都更像自主系统。

核心：

- 草会吸引羊
- 羊数量由草量约束
- 下雨有可见反馈
- 旅人会欣赏世界
- 旅人可以提出交易需求
- 普通旅人满足 2 次需求后离开

# 1. 草吸引羊

只要地图上存在草皮，就有可能吸引新的羊到场。

但羊的数量不能无限增长。

当前 MVP 规则：

每 2 块草皮最多支持 1 只羊。

例如：

- 1-2 块草皮：最多 1 只羊
- 3-4 块草皮：最多 2 只羊
- 5-6 块草皮：最多 3 只羊

这样下雨生成草皮后，会带来羊，但羊也会逐渐消耗草皮，避免全地图铺满草。

# 2. 下雨反馈

下雨时：

- 池塘补满
- 随机生成 3 块草皮
- 地图上出现短暂雨滴动画

# 3. 旅人欣赏

旅人闲逛时会观察可见卡牌。

状态显示为：

正在欣赏 XX

XX 可以是所有非迷雾中的卡牌。

这表示旅人不是无目的随机漂浮，而是在观察这个世界。

# 4. 旅人需求

旅人有两类需求：

## 食物需求

例如：

有没有吃的？

目标：

熟肉

满足方式：

桌子旁 1 格有熟肉，旅人靠近桌子后自动消费。

## 交易需求

旅人可能对可见的非自然资源提出交易：

能把 XX 卖给我吗？

前提：

- 存在桌子
- 目标商品在桌子旁 1 格
- 商品被黄色框和线标记

玩家可以选择：

- 是
- 否

选择“是”：

商品被消费
↓
旅人满足次数 +1

选择“否”：

旅人继续闲逛

# 5. 旅人离开

普通旅人完成 2 次消费/交易后：

离开当前区域。

这代表游客完成一次短暂停留。

# 6. 本轮验收

玩家应能观察到：

- 草多时羊会到来
- 羊会消耗草，控制草的规模
- 下雨产生可见变化
- 旅人会欣赏卡牌
- 旅人会提出交易请求
- 旅人满足 2 次后离开

红色：

否

《MVP数量不决定本质与商品层级 v1.2》

# 1. 数量不决定本质

游戏核心不是数量公式。

数量只影响规模。

本质变化来自：

- 关系
- 加工
- 状态跃迁
- 工序复杂度

因此：

铜钱×3 + 篝火 → 铜块

不是理想规则。

正确规则：

铜钱 + 篝火
↓
铜块

因为铜钱本身就是铜材的货币形态。

进入高热环境后发生本质跃迁。

# 2. 铜饰复杂度

铜饰价值不是来自铜钱数量。

铜饰价值来自多步关系：

铜钱 + 热源 → 铜块

草皮 + 热源 → 干草

玩家 + 干草 → 草绳

草绳 + 铜块 → 铜饰

这符合：

材料 + 条件 + 玩家劳动 + 工艺中介

# 3. 商品定义

桌子周围 1 格有商品时，经营单位进入“有商品”状态。

但不是所有非自然资源都算商品。

当前 MVP 商品标准：

至少经过 2 次加工的非自然卡牌。

例如：

- 熟肉：羊/肉 + 火，属于加工食物
- 铜饰：铜钱/铜块 + 草绳，属于工艺品
- 斧头：碎石塑形 + 木头，属于工具商品

不算商品：

- 肉
- 碎石
- 木头
- 草皮
- 干草
- 草绳
- 铜块

这些是材料或中介物，不是可直接摆摊出售的商品。

《经济锚定：需求、复杂度与铜钱 v1.3》

# 1. 三个经济锚点

《方寸商国》的经济不能只锚定某一种物品。

当前经济系统使用三个锚点：

## 需求锚

旅人是否真的想要。

需求决定：

商品能不能成交。

## 复杂度锚

商品从世界关系中产生，需要经历多少状态跃迁和工序。

复杂度决定：

商品为什么不应该便宜。

## 铜钱锚

铜钱不是价值本身。

铜钱是成交后的记账物。

铜钱代表：

一次外部需求被内部世界满足后留下的信用痕迹。

# 2. 三者关系

复杂度回答：

这个东西凭什么值钱？

需求回答：

为什么有人愿意买？

铜钱回答：

成交后价值如何被记录？

因此：

价值不是单一公式。

而是：

需求 × 复杂度 × 场景适配。

# 3. 复杂度不是数量

复杂度不是：

3 个铜钱

4 个木头

10 个石头

复杂度是：

- 本质跃迁次数
- 参与关系数量
- 是否需要环境条件
- 是否需要玩家实体参与
- 是否需要中介物

例如：

铜钱 + 篝火 → 铜块

不是因为铜钱数量多。

而是铜钱进入热源后发生本质跃迁。

# 4. 工序深度

当前使用“工序深度”作为复杂度的 MVP 表达。

## 深度 0：自然或基础资源

例：

- 木头
- 石头
- 草皮
- 池塘水

特点：

不能直接成为高价值商品。

## 深度 1：一次加工物

例：

- 碎石
- 肉
- 铜块
- 干草

特点：

通常是材料或中介物。

## 深度 2：二次加工物

例：

- 三角碎石
- 正方碎石
- 熟肉
- 草绳
- 木构件

特点：

开始具备商品潜力。

但是否可出售，还要看需求和用途。

## 深度 3：工具 / 结构商品

例：

- 斧头
- 锄头
- 锤子
- 桌子

特点：

具备明确功能或经营价值。

## 深度 4：工艺品 / 高附加值商品

例：

- 铜饰

特点：

由多条加工链汇合而成。

可产生高于材料本身的交易价值。

# 5. 商品判定原则

商品不是所有可移动卡牌。

商品至少需要：

- 非自然资源
- 经过加工
- 能被旅人需求识别
- 适合放在经营单位附近交易

当前 MVP 商品门槛：

复杂度 ≥ 2

但并非所有复杂度 ≥ 2 的东西都一定是商品。

例如：

草绳复杂度为 2，但当前主要是工艺中介物。

# 6. 价格原则

当前不能把价格写死成简单数量公式。

价格应参考：

- 需求强度
- 工序深度
- 稀缺程度
- 场景适配

MVP 占位：

- 熟肉：基础需求商品，1 铜钱
- 工具：功能商品，后续可高于熟肉
- 铜饰：工艺品，当前 5 铜钱

注意：

这些不是最终经济真理。

只是测试用的价格样本。

# 7. 长期方向

后续经济系统应逐步形成：

需求锚：

旅人/居民/商队真正需要什么。

复杂度锚：

世界生产该物需要多少关系和工序。

信用锚：

这个商国是否持续满足外部需求。

货币只是这些关系的记录形式。

《玩家卡、员工智能与上帝视角归因 v1.4》

# 1. 核心定位

《方寸商国》不是纯上帝模拟。

也不是传统角色扮演。

玩家拥有上帝视角操作能力。

但所有上帝视角操作，都必须能在世界内部被归因。

归因对象是：

玩家卡。

因此：

玩家卡是玩家在世界中的化身。

玩家不是无形 UI。

玩家是：

- 世界中的责任主体
- 所有复杂介入的归因点
- 员工智能无法处理之事的最终执行者

# 2. 玩家卡的意义

玩家卡代表：

- 玩家在世界里的身体
- 玩家对世界关系的直接介入
- 复杂问题的处理能力
- 新 combo 的发现和实施

玩家可以从上帝视角拖拽、叠放、砸击、组合卡牌。

但世界内部应理解为：

玩家卡正在安排、推动、加工、干预和解决问题。

# 3. 玩家卡也有人的需求

玩家卡不是纯工具。

玩家卡也应逐步拥有人的需求。

例如：

- 饥饿
- 疲劳
- 安全
- 休息
- 舒适

这些需求不应过早压迫玩家。

但长期必须存在。

因为玩家不是神。

玩家是这个世界中最强的责任主体。

# 4. 员工卡

员工卡是新的局部智能。

员工不是玩家的 UI 按钮。

员工是：

- 世界中的人物卡
- 有需求的自主单位
- 能处理固定领域事务的智能体

员工可以通过被满足生活和工作条件，承担某个领域的重复事务。

例如：

- 采集
- 加工
- 烹饪
- 上架
- 服务
- 巡逻
- 清理

# 5. 员工智能边界

员工只能处理：

- 已经被玩家建立过的稳定链条
- 规则清晰的岗位任务
- 条件满足的重复事务

员工不能处理：

- 新的关系发现
- 复杂 combo
- 异常危机
- 多系统冲突
- 战略性建设
- 未定义需求

这些必须由玩家卡负责。

# 6. 玩家与员工的分工

早期：

玩家亲手处理几乎所有事情。

中期：

员工接管稳定重复事务。

后期：

玩家主要处理：

- 发展问题
- 安全问题
- 管理问题
- 需求变化
- 生态失衡
- 经营瓶颈
- 员工无法理解的新关系

这形成成长曲线：

唯一行动者
↓
责任中心
↓
复杂问题解决者
↓
商业生态治理者

# 7. 游戏核心循环

系统自运行
↓
出现瓶颈 / 风险 / 机会
↓
玩家用上帝视角发现问题
↓
玩家卡通过卡牌关系介入
↓
世界获得新条件
↓
员工和生态继续自运行

这就是《方寸商国》的核心循环。

不是玩家重复劳动。

也不是玩家只看自动化。

而是：

玩家不断发现系统无法自解的问题，并通过卡牌关系解决。

# 8. 生态与经济的关系

生态系统负责提供：

- 生命条件
- 原材料
- 生物循环
- 环境压力

经济系统负责提供：

- 外部需求
- 经营压力
- 信用输入
- 价值记录

两者不是互相抢戏。

两者共同构成：

世界自运行的底盘。

玩家负责处理：

生态和经济自运行中无法自行解决的复杂关系。

# 9. 通商渠道原则

外部世界不应成为传统商店。

外部主要输入：

- 需求
- 信用
- 信息
- 订单
- 客流
- 技能
- 关系机会

外部不应凭空输入内部世界无法涌现的物质。

如果外部带来东西，也必须能被解释为：

- 人带来的知识
- 商队带来的订单
- 信用带来的机会
- 关系网络的扩展

不是：

点击商店购买凭空资源。

# 10. 隐患系统原则

隐患不是为了惩罚玩家。

隐患是：

自动智能无法自行解决的关系问题。

例如：

- 员工饿了，不再工作
- 雨后道路泥泞，旅人到不了桌子
- 草太多，羊太多，生态失衡
- 干草靠近火源，存在火灾风险
- 池塘干涸，水草链中断
- 顾客提出员工无法处理的特殊订单
- 商品链中缺少中介物

玩家解决隐患的方式必须仍然是：

- 拖拽
- 叠放
- 砸击
- 组合
- 改造环境
- 建立新关系

不能只是：

点击按钮扣钱解决。

# 11. 制作纲领

《方寸商国》是一个：

以玩家卡为责任主体，
用上帝视角编排世界关系，
让生态、员工、经营与外部需求逐步自运行的
商业生态卡牌游戏。

玩家的乐趣不是重复执行。

玩家的乐趣是：

- 观察世界行为
- 发现系统瓶颈
- 理解需求和条件
- 编排卡牌关系
- 让世界获得新的自运行能力

# 12. 后续设计优先级

在这个原则下，后续系统应按顺序验证：

## 第一阶段：帮手 / 员工雏形

外部旅人被满足后，有机会留下。

玩家满足其生活与工作条件。

其变成帮手。

帮手承担一个简单岗位。

## 第二阶段：岗位

岗位不是菜单。

岗位是世界关系：

人 + 工具 + 工作点 + 需求
↓
岗位行为

## 第三阶段：隐患

隐患推动玩家使用上帝视角解决复杂问题。

## 第四阶段：通商

通商引入订单、客流、信用和关系机会。

而不是传统外部商店。

《一级食物、草棚与蘑菇大棚 v1.5》

# 1. 为什么不是浆果

一级食物不能太像传统生存游戏里的浆果采集。

《方寸商国》的一级食物应该来自关系链。

因此选择：

蘑菇。

蘑菇不是直接采集。

而是由：

水分 + 木材 + 时间

共同产生。

# 2. 食物层级

## 一级食物

当前代表：

蘑菇

定位：

- 玩家基础口粮
- 员工基础口粮
- 可维持基本工作
- 满足度较低

一级食物的作用不是制造紧张感。

它的作用是：

让玩家和员工有一个可管理、可维持、可自动化的基础生存条件。

## 二级食物

当前代表：

熟肉

定位：

- 更久不饿
- 可提供额外满足感
- 更适合作为旅人商品
- 可用于高级员工或特殊需求

# 3. 湿木头链

木头 + 一桶水
↓
湿木头

湿木头只要不被火烤干：

现实 30 秒
↓
游戏 30 分钟
↓
长蘑菇的木头

这不是简单配方。

这是：

木材在湿润状态下发生菌类生长。

# 4. 采蘑菇

任意人物卡可以对长蘑菇的木头进行左键物理操作。

规则：

第一次砸：

红色数字 1

第二次砸：

红色数字 2
↓
掉落蘑菇×1

任意人物包括：

- 玩家
- 员工

这为员工自动采蘑菇提供基础。

# 5. 草棚

草棚是最基础的居住单位。

关系：

木头 + 草皮
↓
草棚

草棚不是商品。

草棚是生活设施。

作用：

- 居住基础
- 员工生活条件
- 篝火保护条件

# 6. 草棚保护篝火

草棚在篝火旁边 1 格内：

篝火在下雨时不会熄灭。

如果篝火附近没有草棚保护：

下雨
↓
篝火 → 木炭

这让雨天成为经营隐患，而不是纯惩罚。

玩家可以通过布置草棚解决。

# 7. 蘑菇大棚

关系：

草棚 + 长蘑菇的木头
↓
蘑菇大棚

蘑菇大棚是基础食物生产单位。

定位：

- 支撑玩家和员工的基础口粮
- 第一种适合员工管理的生产单位
- 连接居住、食物、木材、水分和劳动

# 8. 蘑菇大棚产量原则

蘑菇大棚不能制造过强生存压力。

它应当：

- 产量足够玩家生存一段时间
- 能支撑早期 1-2 名员工的基础口粮
- 不要求玩家频繁补木头
- 不会让玩家一直陷入喂养焦虑

当前设计倾向：

一次补入木头后，蘑菇大棚应产生多次蘑菇产出。

例如：

木头×1
↓
补入蘑菇大棚
↓
提供若干次采蘑菇机会

具体数值后续试玩调整。

# 9. 木头消耗原则

蘑菇大棚需要木头作为基质。

但木头不能被消耗得过快。

原因：

木头同时承担：

- 工具链
- 结构链
- 草棚
- 桌子
- 蘑菇生产
- 后续建筑

如果蘑菇大棚过快消耗木头，会导致：

后端生产压垮森林。

当前原则：

蘑菇大棚的木头消耗应当缓慢、可预期、有缓冲。

它是维护成本。

不是持续扣资源惩罚。

# 10. 员工第一岗位

员工第一岗位建议为：

照看蘑菇大棚。

岗位行为：

- 检查蘑菇大棚是否有产出
- 采摘蘑菇
- 将蘑菇放到食物区
- 在蘑菇大棚缺基质时补入木头
- 自身饥饿时优先吃蘑菇

员工并不理解新 combo。

员工只执行玩家已经建立好的稳定链条。

# 11. 森林逻辑前置问题

一旦蘑菇大棚消耗木头，森林系统必须被认真设计。

否则会出现两个极端：

## 木头越来越少

如果树木不能再生：

木头会供不应求。

蘑菇大棚、建筑、工具都会互相抢资源。

经济系统会被基础资源瓶颈卡死。

## 木头过于廉价

如果树木生成太快或太多：

木头失去经营意义。

蘑菇大棚维护成本消失。

结构和建筑也会变廉价。

# 12. 森林系统必须讨论的问题

后续必须专门讨论：

- 树木如何再生
- 树苗是否存在
- 下雨是否促进树苗成长
- 草皮/湿地是否影响树木成长
- 砍树是否留下树桩
- 树桩是否能再生
- 员工是否能种树或照看树苗
- 森林是否有生态容量
- 树木再生速度如何避免过快或过慢

森林系统必须既能支撑长期经营，又不能让木头失去价值。

《河流、水文与文明起源 v1.6》

# 1. 核心设定

无论地图如何生成，世界中永远存在一条河流。

河流贯穿地图。

河流不是普通资源点。

河流是：

- 世界水文主轴
- 生态母体
- 文明起源条件
- 后期治理对象
- 通商与交通的潜在基础

玩家不是在任意荒地上凭空建立商业帝国。

玩家是在一条河流滋养的野地上，逐步组织生态、生产、居住、经营和通商。

# 2. 河流的设计意义

河流回答：

为什么这里能发展？

答案：

因为这里有水。

水带来：

- 草地
- 森林
- 动物
- 食物
- 定居
- 交通
- 风险
- 商业机会

因此河流是世界的底层秩序。

# 3. 河流层级

河流属于地形层。

特点：

- 不可搬运
- 不可堆叠
- 可被利用
- 可被治理
- 影响周围空格属性

河流不是卡牌。

但河流会持续影响卡牌和地块。

# 4. 河岸带

河流两侧 1-2 格形成河岸带。

河岸带是重要生态区域。

河岸带可能具有：

- 湿润
- 肥沃
- 泥泞
- 易生草
- 易长树
- 易吸引动物

玩家的选址会因此产生差异：

靠近河流：

- 资源丰富
- 水分充足
- 生态恢复快
- 但风险更高

远离河流：

- 更稳定
- 更安全
- 但需要人工引水和维护

# 5. 河流与森林

森林不应该随机刷新。

森林主要围绕河岸分布。

规则方向：

河岸带
↓
树木更容易生成
↓
树桩更容易再生
↓
木头成为可持续但受条件限制的资源

河流为森林提供水分。

森林为商业帝国提供：

- 木头
- 草棚
- 桌子
- 工具柄
- 蘑菇大棚基质
- 建筑材料

# 6. 河流与草地

草皮也不应该完全随机生成。

草皮应更容易出现在：

- 河岸带
- 雨后湿润区域
- 人工浇水区域

草地支撑：

- 羊
- 草绳
- 草棚
- 生态容量

因此：

河流 → 草地 → 羊 → 肉 → 熟肉 → 旅人消费

形成基础经济链。

# 7. 河流与蘑菇

蘑菇系统也可受河流影响。

湿润环境有利于：

- 湿木头保持湿润
- 长蘑菇的木头形成
- 蘑菇大棚稳定产出

但过湿也可能带来：

- 腐烂
- 泥泞
- 建筑受潮

这为后期管理提供空间。

# 8. 河流水位

河流应有水位系统。

当前设计方向：

- 低水位
- 正常水位
- 高水位

## 低水位

影响：

- 草地生成减少
- 树木再生变慢
- 取水受限
- 鱼类减少

## 正常水位

影响：

- 生态稳定
- 河岸草木正常生长
- 取水稳定

## 高水位

影响：

- 河岸更湿润
- 草地可能快速生长
- 泥地增加
- 道路变慢
- 树木可能烂根
- 建筑可能受潮

高水位不是单纯惩罚。

它是治理问题。

# 9. 河流治理

后期玩家需要治理河流。

可能设施：

- 水渠
- 排水沟
- 堤岸
- 小桥
- 蓄水池
- 水车
- 码头

这些设施不是装饰。

它们改变：

- 水分分布
- 交通效率
- 生态恢复
- 洪水风险
- 通商能力

# 10. 河流与取水

空桶可以从河流取水。

河流不同于池塘。

池塘是点状水源。

河流是线状水文系统。

后续池塘可以成为：

- 河流支流
- 河湾
- 蓄水点
- 人工水池

当前 MVP 可以先将池塘逻辑逐步迁移为河流逻辑。

# 11. 河流与捕鱼

河流可以捕鱼。

但捕鱼不能成为初期稳定生活来源。

规则原则：

玩家可以钓鱼。

但：

- 很慢
- 不稳定
- 产量低
- 不足以支撑长期生活

规模化捕鱼需要：

- 工具
- 渔网
- 员工岗位
- 码头
- 时间

这样鱼是补充资源，不会取代蘑菇、羊和经营链。

# 12. 河流与通商

河流后期可以成为通商渠道。

不是传统商店。

而是：

- 船只到来
- 客流增加
- 订单出现
- 外部消息进入
- 商队路线形成
- 信用网络扩展

河流让外部世界有了进入内部世界的自然路径。

# 13. 河流风险

河流带来资源，也带来风险。

可能风险：

- 洪水
- 泥泞
- 烂根
- 建筑受潮
- 道路阻断
- 水位过低
- 河岸退化

这些风险不是为了惩罚玩家。

它们是：

自动系统无法自解的水文关系问题。

玩家需要通过卡牌关系和设施治理。

# 14. 与玩家目标的关系

玩家最终不是简单经营摊位。

玩家是在治理一条河流滋养的商业生态。

成长路径：

河边野地
↓
基础取水
↓
草地与森林
↓
蘑菇/羊/木材生产
↓
桌子与旅人
↓
居住与员工
↓
道路与设施
↓
河流治理
↓
通商网络
↓
商业帝国

# 15. 制作原则

河流不是背景。

河流是规则。

所有与生态和商业相关的系统，都应逐步回答：

它与河流有什么关系？

例如：

- 树为什么长在这里？
- 草为什么在这里出现？
- 羊为什么来这里？
- 人为什么能定居？
- 客人为什么能到来？
- 商队为什么能通行？
- 风险为什么发生？
- 玩家为什么需要治理？

河流是世界关系的底层纽带。

《卡牌涌现性校准 v1.8》

# 1. 核心修正

玩家不是在搬资源。

玩家是在调世界关系。

《方寸商国》不能变成：

- 生存采集游戏
- 资源挖掘游戏
- 工具等级解锁游戏
- 套着卡牌皮的建造游戏

卡牌的本质不是“背包物品”。

卡牌的本质是：

封装后的世界关系节点。

# 2. 什么是卡牌涌现性

卡牌涌现性不是“更多配方”。

而是同一张卡在不同关系中产生不同意义。

例如树不是木头矿。

树至少可以是：

- 河岸生态节点
- 观赏对象
- 遮阴来源
- 树枝掉落来源
- 木材放大来源
- 河岸稳定因素
- 火灾风险对象
- 过度开发后的生态压力源

木头只是树参与某些关系后的产物之一。

不是树的全部意义。

# 3. 自然物不是库存

河流不是桶装水库存。

树不是木头库存。

草不是羊饲料库存。

自然物应是持续参与关系的世界节点。

玩家可以利用自然物。

但不能把自然物理解成固定数量的箱子。

# 4. 河流修正

河流属于世界轴。

空桶可以从河流取水。

但桶不能把整条河流抽干。

取水的影响应表现为：

- 局部河岸压力
- 湿润度变化
- 后续水位管理需求

而不是：

河流可取水量 3 / 3

河流是流动系统。

池塘才可以是有限水体。

# 5. 树木修正

树不能只靠斧头砍。

否则玩家会进入传统生存游戏逻辑：

没有斧头 → 没有木头 → 必须做斧头 → 砍树。

这不是本项目的核心。

树应改为：

扎根生态节点。

基础规则：

- 树不可拖拽
- 树会周期性掉落树枝
- 树枝是早期木质关系入口
- 斧头砍树可以快速获得木头
- 但会让树进入受损状态
- 受损树暂停自然掉枝

这样斧头不是资源门槛。

斧头是效率工具，也是生态代价工具。

# 6. 树枝定位

树枝不是“更小的木头数值”。

树枝是自然掉落物。

它的意义是：

- 火源维护
- 简易结构
- 木构件入口
- 早期建造关系
- 后续蘑菇基质的基础材料之一

树枝可以参与木质关系。

但它的定位应弱于完整木头。

树枝让玩家能在不砍树的情况下开始编排世界关系。

# 7. 斧头定位

斧头不应是“解锁木头”的钥匙。

斧头应是：

- 提高效率
- 造成破坏
- 产生生态代价
- 支撑规模化生产

玩家可以不用斧头也获得木质材料。

但如果玩家追求更快发展，就会考虑是否伤树。

这让工具选择变成经营判断，而不是线性科技门槛。

# 8. 本轮 MVP 调整

当前 MVP 先做三件事：

## 河流

空桶从河流取水不再让河流变干。

连续取水只增加“河岸压力”显示。

雨水和时间会缓慢恢复河岸压力。

## 树

树保留扎根状态。

树周期性掉落树枝。

树枝掉在树旁空格。

## 斧头

斧头砍树不再删除树。

斧头会让树进入受损状态，并掉落更多木头。

受损树在一段时间内不再自然掉枝。

# 9. 试玩目标

玩家应能感受到：

- 自然不是库存
- 河流不是水桶仓库
- 树不是木头矿
- 不砍树也能从自然掉落进入木质关系
- 砍树是高效率但有代价的选择

这一步的目标不是完善森林系统。

而是把游戏重新拉回：

关系驱动的生态经济卡牌世界。

《卡牌关系抽象层 v1.9》

# 1. 为什么需要关系抽象层

《方寸商国》的卡牌不是资源块。

也不是配方材料。

卡牌是世界关系的可见节点。

如果没有清晰的关系抽象层，游戏很容易滑向：

- 采集资源
- 合成工具
- 解锁科技
- 堆数量
- 做库存

这不是本项目的核心。

本项目的核心是：

玩家通过改变关系、建立关系、诱导关系，让世界向自己的目标发展。

因此，每个新增系统都必须回答：

- 它属于哪些关系层？
- 它改变了哪条已有关系链？
- 它给玩家提供了什么插入变量的机会？
- 它是否让世界更会自己运行？

# 2. 八个基础关系层

当前将卡牌世界的关系抽象为八层。

这八层不是互斥分类。

一张卡、一个事件、一个系统，往往同时属于多层关系。

## 2.1 并列关系

并列关系指：

两张或多张卡牌因为空间接近而产生意义。

它们不融合。

但它们因为相邻、附近、围绕、夹住、贴近而进入新关系。

例子：

桌子旁边 1 格有熟肉。

结果：

熟肉进入可交易状态。

并列关系适合表达：

- 展示
- 经营
- 仓储
- 保护
- 居住
- 工作区
- 商品区
- 服务区

并列关系是卡牌游戏最直观的空间语言。

## 2.2 叠合关系

叠合关系指：

两张卡牌通过右键叠放或指定组合，改变本质。

例子：

木构件 + 木头
↓
桌子

铜块 + 草绳
↓
铜饰

叠合关系不是数量公式。

它表达的是：

两个世界单位建立了新的本质关系。

叠合关系适合表达：

- 加工
- 组合
- 结构形成
- 工艺制作
- 本质跃迁

原则：

叠合不是“凑够数量”。

叠合是“关系足以改变本质”。

## 2.3 场域关系

场域关系指：

某张卡或地形向周围产生持续影响。

例子：

篝火产生热源。

草皮在热源范围内逐渐变成干草。

河流影响河岸湿润。

场域关系适合表达：

- 热
- 湿
- 光
- 安全
- 危险
- 污染
- 舒适
- 腐败
- 肥沃

场域关系让卡牌不必直接接触，也能互相影响。

它是生态系统和居住系统的基础。

## 2.4 时间关系

时间关系指：

某个关系不是瞬间完成，而是经过一段时间成熟、衰退、腐解或转化。

例子：

湿木头保持湿润 30 秒。
↓
长蘑菇的木头。

羊尸体停留 15 秒。
↓
腐解，生成草皮和灌木。

羊吃草 90 秒。
↓
草皮消失。

时间关系的作用：

- 让世界有生命感
- 给玩家操作窗口
- 让自动系统能运行
- 让风险和机会自然出现

时间关系不能只是等待条。

它应该连接到生态、经济和角色行为。

## 2.5 行为关系

行为关系指：

有能动性的卡牌主动寻找、靠近、逃离、消费、攻击、工作或沟通。

例子：

羊寻找草。

狼追羊。

旅人靠近可见商品。

玩家卡饿了会寻找食物。

员工根据岗位关系工作。

行为关系适合表达：

- 生物
- 人物
- 旅人
- 员工
- 客户
- 危险体

行为关系的原则：

人物和生物不是 UI 动画。

它们是世界中的行动者。

但它们也不是无限智能。

它们只在自己能感知到的关系场中做有限选择。

## 2.6 状态关系

状态关系指：

同一张卡牌因为状态不同，而接入不同关系链。

例子：

羊：

- 活着
- 眩晕
- 尸体
- 腐解中

树：

- 扎根
- 受损
- 枯死
- 结果

河岸：

- 湿润
- 干岸
- 泥泞
- 肥沃

状态关系的重点：

卡牌类型不一定改变。

但它的关系资格改变。

活羊可以吃草、逃跑、被狼追。

眩晕羊可以被屠宰。

羊尸体可以腐解、吸引危险、产生卫生问题。

状态关系让世界更细腻。

也避免所有变化都必须变成新卡。

## 2.7 所属关系

所属关系指：

某张卡、某个区域、某个岗位或某个商品，被纳入某个责任结构。

它不一定依赖空间邻近。

例子：

这张桌子属于经营点。

这个员工属于蘑菇大棚岗位。

这张商品属于可销售区。

这个草棚属于某个人的住所。

所属关系适合表达：

- 店铺
- 岗位
- 住所
- 责任区
- 商品归属
- 员工职责
- 客户订单
- 商队协议

没有所属关系，后期商业帝国会散。

所属关系让玩家从手工执行者，变成关系组织者。

## 2.8 信号关系

信号关系指：

某张卡或某个关系，不直接转化另一张卡，但会向世界发出吸引、排斥、提示或诱导。

例子：

草皮存在。
↓
吸引羊。

桌边有商品。
↓
吸引旅人。

狼靠近。
↓
让羊和人产生恐惧。

篝火 + 草棚。
↓
形成定居信号。

信号关系是“玩家不能命令人，只能创造条件”的关键。

玩家通过布置信号，诱导世界行动。

信号关系适合表达：

- 吸引
- 排斥
- 威胁
- 机会
- 需求
- 招募
- 客流
- 居住意愿
- 商业繁荣度

# 3. 八层关系的组合

八个关系层通常会组合出现。

例如：

桌子旁边有熟肉。

这是并列关系。

桌子旁边的熟肉发出商品信号。

这是并列关系 + 信号关系。

旅人接收到商品信号，靠近桌子提出购买。

这是信号关系 + 行为关系。

交易完成后留下铜钱。

这是行为关系 + 经济关系。

铜钱放进火源后变成铜块。

这是叠合关系 + 场域关系。

铜块和草绳变成铜饰。

这是叠合关系 + 工艺复杂度。

铜饰放在桌边吸引旅人。

这是并列关系 + 信号关系。

这就是《方寸商国》的基本生成方式：

不是资源公式。

而是关系层不断叠加。

# 4. 玩家操作的重新定位

玩家仍然可以左键和右键操作。

但它们不应承担大量重复劳动。

左键主要表达：

- 强干预
- 打断
- 破坏
- 砸击
- 驱赶
- 救急

右键主要表达：

- 建立关系
- 指定用途
- 授权交易
- 组合本质
- 创建所属

频繁重复的日常劳动，不应该交给上帝视角玩家。

它应该交给：

- 世界自动关系
- 玩家卡自动执行
- 员工岗位行为

玩家真正做的是：

观察关系。

插入变量。

建立条件。

改变流向。

# 5. 玩家卡与能动性

玩家卡不是摄像机。

玩家卡是玩家在世界中的化身。

但玩家也不能完全命令玩家卡的生活逻辑。

玩家卡有自己的需求和能动性。

它会：

- 饿了找食物
- 有危险就逃离或防御
- 客人来了尝试沟通
- 有可执行任务就去处理
- 忙不过来时暴露瓶颈

玩家能做的是创造条件。

例如：

玩家不能直接命令：

你现在去做饭。

但玩家可以建立：

火源 + 食材 + 可用工具 + 安全空间。

玩家卡会在需要时尝试处理。

# 6. 与传统管理游戏的区别

本项目会与《环世界》等游戏在外观上有相似之处：

- 有自动人物
- 有需求
- 有岗位
- 有危险
- 有生产

但底层目标不同。

传统管理游戏常见核心是：

- 资源库存
- 工作优先级
- 科技树
- 数值建设
- 大量物资消耗

《方寸商国》的核心应是：

- 关系建立
- 关系改变
- 状态转向
- 信号诱导
- 生态反馈
- 经济涌现

玩家不是直接管理殖民地运转。

玩家是创造关系条件，让世界自己决定能不能运转。

# 7. 后续制作铁律

以后每新增一张卡、一个系统、一条规则，都必须回答：

它至少涉及哪一种关系层？

它是否只是资源公式？

它是否能让世界自己多发生一件事？

它是否给玩家提供了插入变量的机会？

它是否能被角色自动感知或使用？

如果答案是否定的，要谨慎加入。

因为它可能只是传统生存/经营游戏的资源链。

而不是《方寸商国》的关系链。

《能动性与自动生态链 MVP v2.0》

# 1. 本轮目标

验证一个关键体验：

世界本来就在运行。

玩家只是插入变量，改变关系流向。

玩家不应该频繁左键敲、右键叠，像上帝视角农民一样执行所有劳动。

频繁劳动应该由：

- 世界自动关系
- 玩家卡自身能动性
- 后续员工岗位

来承担。

# 2. 玩家卡最小能动性

玩家卡不是摄像机。

玩家卡是玩家在世界中的化身。

本轮先实现一个最小需求：

饥饿。

规则方向：

玩家卡会逐渐饥饿。

如果附近或可达区域存在可吃食物：

玩家卡会自己走过去吃。

如果没有食物：

玩家卡会表达需求：

我需要食物。

玩家不能直接命令玩家卡生活。

玩家能做的是创造条件：

- 提供食物
- 建立生产关系
- 保证安全
- 保证路径可达

这验证“创造条件，而不是直接命令”的核心。

# 3. 世界自动关系链

本轮加入最小草羊狼尸体链。

基础链条：

草
↓
吸引羊
↓
羊吃草
↓
狼追羊
↓
羊死亡变尸体
↓
尸体腐解
↓
草皮增加 + 灌木生成

这条链不需要玩家启动。

玩家进入世界时，应该能看到它自己运行。

# 4. 玩家插入变量

玩家可以通过强干预改变关系流向。

例子：

世界原本：

草 → 羊 → 狼

玩家插入：

石头砸羊。

结果：

羊进入眩晕状态。

如果继续砸：

羊死亡，变成羊尸体。

如果屠宰：

进入食物生产链。

如果不处理：

进入腐解生态链。

这就是关系改变。

重点不是“掉落什么资源”。

重点是：

一张羊卡从生态链切换到了腐解链或食物链。

# 5. 羊尸体

羊尸体不是肉。

羊尸体是一个过渡状态。

它可以：

- 被屠宰
- 腐解
- 吸引后续危险
- 影响卫生
- 滋养草地

当前 MVP 先做：

羊尸体停留 15 秒。

然后腐解为：

- 草皮 ×3
- 灌木丛 ×1

后续再加入：

- 腐败气味
- 卫生风险
- 食腐动物
- 员工清理岗位

# 6. 狼

狼是自动生态关系的一部分。

狼不是给玩家刷怪的惩罚。

狼的作用是让羊不再只是“草的消费者”，而是生态链中的中间节点。

狼会：

- 感知羊
- 追羊
- 咬死羊
- 让羊进入尸体链

后续狼可扩展为：

- 威胁旅人
- 威胁玩家卡
- 被火源驱离
- 被围栏阻挡
- 被员工或工具处理

# 7. 灌木丛

灌木丛是腐解后的生态反馈。

当前 MVP 先作为自然生态节点存在。

后续可扩展为：

- 遮蔽
- 小动物栖息
- 旅人观赏
- 浆果/蘑菇关系
- 火灾风险
- 河岸生态稳定

# 8. 本轮验收

玩家应能看到：

- 玩家卡有自己的饥饿状态
- 有食物时玩家卡会自己去吃
- 没食物时玩家卡会表达需求
- 狼会追羊
- 羊会变成尸体
- 尸体会腐解成草和灌木

最重要的试玩感受：

世界不是等待玩家合成。

世界已经在运行。

玩家是在插入变量，改变关系。

《生活逻辑修正包 v2.1》

# 1. 本轮问题

上一轮 MVP 已经证明：

世界可以自己跑。

但试玩后出现新的问题：

世界会动，但还不够会生活。

典型问题：

- 狼吃饱后仍停在羊身边
- 羊看见狼没有明显回避
- 草、石头、树枝等小物会阻塞生物行动
- 玩家卡饿了只会找现成食物或喊饿
- 灌木丛还没有基础采集定位

这些问题会让自动关系显得机械。

本轮目标是让生态关系更像生活逻辑。

# 2. 阻挡层级

卡牌世界中的“物理阻挡”需要分层。

上帝视角拖拽时，卡牌之间仍可发生左键物理碰撞。

但生物和人物寻路时，不应把所有卡都当墙。

## 不阻挡生物/人物寻路

- 草皮
- 灌木丛
- 石头
- 碎石
- 树枝
- 干草
- 草绳
- 铜钱
- 小型材料

这些是低矮物、小物、地面物。

生物和人物可以经过。

## 阻挡生物/人物寻路

- 树
- 河流
- 篝火
- 桌子
- 大型动物
- 人物
- 后续建筑

这些是地形、大型实体、危险源或明确结构。

# 3. 羊的恐惧关系

羊不是只会吃草。

羊应该感知狼。

规则方向：

狼在 3 格内。
↓
羊停止吃草。
↓
羊进入逃离状态。
↓
优先远离狼。

狼远离后，羊恢复找草。

这让羊从“草的消费者”变成真正的生态行动者。

# 4. 狼窝关系

狼吃饱后不应在羊群旁休息。

狼需要回到休息关系。

狼窝的本质不是凭空建筑。

狼窝的本质先定义为：

干草形成的巢穴条件。

当前 MVP 简化：

干草被狼识别为狼窝。

狼捕食后进入饱腹状态。

饱腹狼会优先回到干草附近。

如果没有干草，则寻找远离羊、玩家和篝火的位置休息。

后续可以扩展为：

干草 + 灌木
↓
狼窝

或：

干草在灌木/树附近持续一段时间
↓
形成狼窝状态。

# 5. 玩家卡采集逻辑

玩家卡饿了不应该原地说“我饿了”。

这只能作为失败反馈。

玩家卡应按优先级尝试生活：

1. 吃现成可食物
2. 采集灌木
3. 简单加工可食物
4. 使用已有工具捕猎
5. 条件足够时尝试制作基础工具
6. 仍无法解决才表达需求

当前 MVP 先做前两步：

有蘑菇或熟肉。
↓
玩家卡自己去吃。

没有现成食物，但有灌木丛。
↓
玩家卡自己去采集。
↓
灌木丛掉落蘑菇。
↓
玩家卡再去吃蘑菇。

这样灌木丛成为基础采集生态位。

# 6. 灌木丛生态定位

灌木丛不是装饰。

它是中层生态节点。

当前定位：

- 可采集基础食物
- 可由尸体腐解生成
- 可被旅人欣赏
- 可作为狼窝/小动物栖息的条件之一

后续可扩展：

- 浆果
- 蘑菇
- 枝条
- 遮蔽
- 火灾风险
- 过密阻路

# 7. 本轮验收

玩家应能看到：

- 草、石头、树枝等小物不再卡死生物行动
- 羊看到狼会拉开距离
- 狼捕食后会离开羊群，倾向回干草附近
- 玩家卡饿了会先找现成食物
- 没有现成食物时会自己采集灌木
- 灌木采集后会掉落基础食物

这一步的核心不是增加新资源。

而是让已有关系有生活逻辑。

--------------------------------------------------

《可读性、工具携带与定居边界 v2.2》

# 1. 本轮目标

世界运转感已经出现。

下一步要让玩家更容易看懂关系，并让人物更像会利用条件生活。

本轮先落地五个方向：

- 卡牌颜色
- 工具携带
- 火源恐惧
- 草棚基础居住
- 玩家卡最低生活维护的第一步

# 2. 卡牌颜色

卡牌颜色不是美术装饰。

它是测试关系的认知工具。

不同生态位、物质层、危险层应该有不同颜色。

当前颜色原则：

- 草皮：嫩绿
- 干草：枯黄
- 树：深绿
- 灌木：墨绿
- 石头/碎石：灰色
- 木头/树枝：棕色
- 火源：橙红
- 水：蓝色
- 人物：淡蓝
- 动物：浅棕
- 危险动物：暗红
- 尸体/腐解：暗褐或灰紫
- 商品/货币：金黄

玩家应该能不读文字，也大致看出世界关系状态。

# 3. 工具携带关系

工具不是单纯摆在地上的物品。

人物使用工具时，工具应与人物建立携带关系。

规则方向：

工具右键叠到人物卡上。
↓
工具不再占地图格。
↓
人物获得对应能力。

例子：

石制小刀 + 玩家卡
↓
玩家卡具备切割能力。

锤子 + 玩家卡
↓
玩家卡具备敲击能力。

斧头 + 玩家卡
↓
玩家卡具备砍伐能力。

选中人物卡时，应显示携带工具。

工具携带关系是后续自动加工的基础。

# 4. 火源恐惧与文明边界

火不是单纯烹饪工具。

火是文明边界。

狼会追羊，也会追人。

但狼靠近火源时会害怕。

规则方向：

狼追逐目标。
↓
如果目标靠近火源，或狼自己进入火源恐惧范围。
↓
狼放弃追逐，远离火源。

人遇到狼时，也会倾向靠近火源。

这让火成为：

- 安全信号
- 定居中心
- 夜间边界
- 后续营地核心

# 5. 草棚基础居住

草棚是基本居住单位。

关系：

木头或树枝 + 草皮
↓
草棚

草棚靠近篝火 1 格。
↓
形成基础定居关系。

之前规则保留：

草棚在篝火旁 1 格内，篝火下雨不会熄灭。

如果篝火旁没有草棚，下雨后篝火会变成木炭。

当前 MVP 先实现草棚卡和火边定居信号。

# 6. 玩家卡最低生活维护

玩家卡不应等待上帝视角执行所有加工。

但也不能一次性加入过多自动工业。

本轮先做最低层：

- 玩家卡会携带工具
- 玩家卡选中可看到携带工具
- 玩家卡遇到危险会往火边逃
- 玩家卡饿了仍优先吃现成食物、采集灌木

后续再逐步加入：

- 自动制作最低小刀
- 自动制作最低草棚
- 自动整理食物和商品
- 自动维护篝火

# 7. 本轮验收

玩家应能看到：

- 卡牌颜色更容易区分生态和经济关系
- 工具能叠到玩家或人物身上
- 选中人物能看到携带工具
- 狼靠近火会放弃追逐
- 人遇到狼会往火边跑
- 木质材料和草皮能形成草棚

这一步的核心是：

火、工具、住所开始把人物从野外生活推向定居关系。

《行动者差异、生命与狼窝 v2.3》

# 1. 本轮问题

目前羊、狼、人、旅人的行为差异还不够明显。

它们只是目标不同。

但作为行动者，它们应该拥有不同生态性格。

本轮目标：

- 不同生命体有不同速度
- 不同生命体有不同血量
- 左键砸生命体进入伤害系统
- 羊被堵时会突围
- 狼饱腹后会建设狼窝
- 狼窝可被砸回干草
- 旅人遇狼会逃离或避险

# 2. 基础血量

以“砸”为生命交互基础。

当前定义：

- 羊：2 血
- 狼：4 血
- 人：6 血
- 旅人：4 血

生命体被砸会扣血。

血量归零后：

羊 → 羊尸体。

狼 → 狼尸体或暂时移除。

人 → 受伤/失能。

当前 MVP 先做：

羊血量归零变羊尸体。

狼血量归零逃离。

人和旅人血量归零进入受伤状态。

# 3. 加工与伤害分离

砸生命体是伤害。

砸材料是加工。

例如：

石头砸石头。
↓
碎石。

石头砸羊。
↓
羊受伤。

这两套逻辑需要分开。

# 4. 羊突围

羊看见狼会逃。

如果被堵在角落，不应该原地来回跳。

羊的突围逻辑：

- 优先远离狼
- 如果最优方向被堵，随机选择可走方向
- 可以短暂牺牲最远距离以摆脱角落

羊不是规划者。

羊是惊慌的动物。

所以突围行为可以带随机性。

# 5. 狼窝

狼不应该吃饱后在羊群旁休息。

狼会建设自己的生态位。

规则方向：

狼饱腹后寻找草皮或干草。
↓
把草/干草带到地图边缘。
↓
形成狼窝。

狼窝的本质是干草的动物占有状态。

狼窝可被砸。

狼窝被砸 2 下：

狼窝 → 干草。

狼窝叠狼：

狼进入潜伏/隐藏状态。

羊无法感知窝中狼。

玩家或人靠近后才可能发现。

# 6. 行动者速度

不同生物的移动频率不同。

当前建议：

- 狼：快
- 羊：中
- 玩家/人：中慢
- 旅人：慢

速度差异通过行动间隔体现。

不是所有卡每个 tick 都移动。

# 7. 旅人危险反应

旅人不是生态参与者。

旅人遇到狼应优先离开或向火边躲避。

当前 MVP：

旅人感知狼后进入受惊。

有火则靠近火。

无火则离开地图。

# 8. 屠宰规则确认

羊尸体被刀屠宰就是肉。

这是食物生产链。

如果不屠宰，羊尸体会进入腐解链。

因此玩家或人物面对羊尸体时，存在两条关系选择：

- 屠宰 → 肉
- 不处理 → 腐解 → 草皮/灌木

这是关系流向选择，不是单纯掉落。

《种群生命周期与供养关系 v2.4》

# 1. 本轮目标

生态不应通过刷新成年个体来维持。

动物繁殖必须进入生命周期关系：

成年个体
↓
幼体
↓
被照料或跟随
↓
成长为成年个体

幼体不是小号成年动物。

幼体有自己的生存逻辑。

# 2. 灌木与浆果

灌木不再产蘑菇。

灌木产出少量浆果。

食物层级暂定：

- 浆果：野外采集，少量，救急
- 蘑菇：湿木头/蘑菇大棚体系，稳定基础食物
- 熟肉：二级食物，更高饱腹与后续加成

这让灌木和蘑菇重新分工。

# 3. 羊的生命周期

成年羊有性别：

- 公
- 母

必须同时存在成年公羊和成年母羊，且草地条件足够，才可能出现羊羔。

羊羔规则：

- 血量 1
- 紧跟成年羊
- 也会吃草，但不是繁殖主体
- 看见狼会逃
- 成长后变成年羊，并随机性别

这让羊群不是刷新，而是繁殖关系。

# 4. 狼的生命周期

成年狼有性别：

- 公
- 母

必须同时存在成年公狼和成年母狼，并且存在狼窝，才可能出现幼狼。

幼狼规则：

- 血量 1
- 待在狼窝
- 不捕猎
- 不繁殖
- 狼窝被破坏时暴露

# 5. 幼狼供养

幼狼不能只靠时间长大。

幼狼必须被成年狼定期喂肉。

规则方向：

成年狼捕食羊。
↓
成年狼携带肉。
↓
成年狼回狼窝。
↓
幼狼获得一次喂养。

幼狼喂养达到 3 次。
↓
成长为成年狼。

这让捕食者数量受制于：

- 羊的数量
- 成年狼捕猎能力
- 狼窝安全
- 肉的供给

如果羊少，幼狼长不大。

如果狼窝被破坏，幼狼暴露且容易死亡。

这就是生态容量。

# 6. 迁移与灭亡

迁移不是刷新。

迁移是外部生态对本地条件的回应。

当前方向：

无羊但草足够。
↓
少量羊迁入。

无狼但羊足够且有遮蔽。
↓
少量狼迁入。

如果没有生存条件，迁入物种仍会持续灭亡或离开。

# 7. 与人物卡交叉验证

人物卡的生存逻辑需要逐步跟上生态复杂度。

当前方向：

- 玩家卡会吃浆果、蘑菇、熟肉
- 玩家卡会采集灌木
- 玩家卡会避狼、靠近篝火
- 后续会维护营地、火、食物余量和基础工具

生态压力必须能推动人物建立生活关系。

《生态负反馈与营地雏形 v2.5》

# 1. 本轮目标

当前生态已经有生命周期，但仍会失衡：

- 狼数量容易滚雪球
- 羊死后草过多
- 羊迁入点不合理
- 玩家卡还没有存储和发展倾向

本轮目标不是增加新配方。

而是增加负反馈和人类营地雏形。

# 2. 狼的负反馈

狼必须受猎物供给限制。

规则方向：

- 狼有饥饿
- 长时间没有捕食会迁出
- 幼狼长时间没有肉会死亡
- 狼窝有容量
- 狼数量超过本地猎物承载时，成年狼会迁出

这让狼不能无限增长。

狼强，会减少羊。

羊少，会让狼缺肉。

狼缺肉，幼狼长不大，成年狼迁出。

# 3. 羊迁入方向

羊迁入不是刷新。

羊迁入是外部生态响应本地草地条件。

规则方向：

- 草足够
- 狼窝远
- 有开阔地

才吸引羊迁入。

羊应从远离狼窝的地图边缘进入。

羊偏好开阔草地，不应从狼窝方向出现。

# 4. 草过密老化

草过多但没有羊利用时，会逐渐老化。

规则方向：

草皮过密。
↓
部分草皮变成干草。

干草可以：

- 被羊吃
- 成为狼窝材料
- 成为草绳/草棚体系材料
- 后续带来火灾风险

这样草不会无限堆积为无意义绿色噪声。

# 5. 营地识别

玩家卡应识别：

篝火 + 草棚
↓
营地

营地不是 UI 按钮。

营地是玩家卡生活逻辑的锚点。

当前 MVP：

- 有火边草棚时，玩家卡认为这里是营地
- 玩家卡倾向把食物搬到营地附近
- 玩家卡倾向把木质材料/小材料搬到营地附近
- 危险时玩家卡倾向回营地

# 6. 工具发展边界

不新增工具合成捷径。

工具仍然遵守既有关系：

坚硬物砸碎石。
↓
塑形。

工具头 + 木质材料。
↓
工具。

玩家卡后续可以自动执行既有关系链。

但不能凭空使用新配方。

当前 MVP 只做：

- 地上已有工具时，玩家卡会拿起
- 没有工具时，玩家卡可以暴露“缺少工具条件”

自动塑形和自动组合后续需要任务队列后再做。

# 7. 本轮验收

玩家应能看到：

- 狼群不会无限爆炸
- 幼狼断粮会死亡
- 羊迁入更远离狼窝
- 草过多会转化为干草
- 火边草棚成为玩家卡营地锚点
- 玩家卡会把食物/材料搬到营地附近

《生态小网修复 v2.6》

# 1. 本轮问题

当前生态仍然是短链。

草 → 羊 → 狼 的链条太直接。

狼没有足够的压力分流。

羊繁殖跟不上狼捕猎。

玩家卡也没有形成真正的营地发展意愿。

本轮目标：

- 限制狼窝容量
- 修复玩家卡避险瞬移
- 让玩家卡更明确地处理营地任务
- 加入小型草食缓冲物种

# 2. 狼窝容量

狼窝不是无限收容点。

当前规则：

一个狼窝最多稳定容纳：

- 成年公狼 1
- 成年母狼 1
- 幼狼 1

超过容量的成年狼会迁出。

这样狼群扩张需要新的狼窝。

# 3. 玩家卡营地任务

玩家卡不能只是吃和躲。

营地形成后，玩家卡应有最低发展意愿：

- 搬食物到营地
- 搬木材/树枝/石头到营地
- 有刀且看见羊尸体，尝试屠宰
- 没有刀时表达缺少工具条件

本轮不新增工具合成捷径。

工具仍然按原有关系链产生。

# 4. 小型草食缓冲物种

生态不能只有羊承担全部食草压力。

新增小型草食动物：

暂名：野兔。

定位：

- 吃草/浆果
- 繁殖快
- 血量 1
- 被狼捕食
- 肉少

野兔的作用不是提供新产业。

而是作为狼捕食压力的缓冲层。

狼不再只盯羊。

草也不再只靠羊消耗。

# 5. 本轮验收

玩家应能看到：

- 狼数量更难爆炸
- 狼窝容量更清晰
- 野兔会消耗草并被狼捕食
- 玩家卡避险不再瞬移
- 玩家卡对尸体/营地物资更有反应

《攻击表达、进食恢复与草地水文 v2.7》

# 1. 本轮目标

当前生态已经有个体和种群，但攻击表达不清晰。

狼碰到人不应直接饱腹。

攻击必须像左键“砸”一样有明确反馈：

- 被攻击者抖动
- 跳出红色数字
- 扣血
- 进入对应状态

本轮目标：

- 狼状态从“追羊”统一为“捕猎”
- 攻击采用冷却制
- 每次攻击有抖动和红色数字
- 人被狼攻击会扣血
- 人有工具时会尝试反击
- 自动卡进食后会回血
- 草地有水文恢复逻辑

# 2. 攻击表达

自动攻击不能只是接触触发结果。

攻击流程：

攻击者靠近目标。
↓
攻击冷却完成。
↓
被攻击者抖动。
↓
跳出红色数字。
↓
扣 1 血。

这与左键砸的表达保持一致。

# 3. 狼捕猎

狼的状态不应叫追羊。

狼的状态是：

捕猎。

目标可以是：

- 野兔
- 羊
- 羊羔
- 玩家卡

狼攻击人不会饱腹。

只有成功捕杀可食猎物，或携肉回窝，才进入食物关系。

# 4. 人被攻击与反击

玩家卡有基础血量。

被狼攻击会扣血。

血量为 0 时进入倒地或重伤状态。

玩家卡如果携带工具：

- 小刀
- 斧头
- 锤子

被狼近身时，会尝试反击一次。

这不是玩家主动攻击，而是人物卡的自保行为。

# 5. 进食回血

所有自动卡进食后都会回血。

规则方向：

羊吃草后恢复生命。

野兔吃草/浆果后恢复生命。

狼捕食或吃肉后恢复生命。

玩家卡吃食物后恢复生命。

进食不是只改变饥饿。

进食也是生命恢复关系。

# 6. 草地水文恢复

植物生成逻辑要更清楚。

草地来源包括：

- 河岸湿润
- 下雨
- 尸体腐解
- 玩家浇水

当前 MVP 先做：

河岸湿润格会周期性尝试长草。

如果附近草太少，河岸更容易生草。

这样草不是纯随机刷新。

草来自水文与腐解。

# 7. 本轮验收

玩家应能看到：

- 狼显示“捕猎”而不是追羊
- 攻击有抖动和红色数字
- 狼攻击人会扣血但不饱
- 人携带工具时会反击
- 吃东西后生命恢复
- 河岸附近会自然恢复草地

《肉、眩晕、果实与人物任务 v2.8》

# 1. 本轮问题

当前暴露出几个根本问题：

- 肉的来源和搬运逻辑不清楚
- 眩晕不是通用机制
- 果实/种子没有繁衍逻辑
- 玩家卡仍然没有足够明确的发展任务

这些不是数值问题。

它们关系到世界是否能被人物卡理解和参与。

# 2. 肉的逻辑

肉不是模糊奖励。

不同动物的肉量不同。

当前定义：

- 野兔 → 肉 ×1
- 羊 → 肉 ×2

狼不能凭空吃饱。

狼必须通过食物关系获得肉。

规则方向：

狼捕杀猎物。
↓
尸体/肉出现。
↓
狼从尸体或肉卡取得 1 张肉。
↓
狼携带 1 张肉回窝或自己吃。

一只狼一次只能携带 1 张肉。

狼吃饱需要 1 张肉。

吃饱后进入消化时间。

消化完成前，狼不继续频繁捕猎。

# 3. 眩晕共性机制

眩晕不是羊的特殊状态。

眩晕是自动卡的通用状态。

规则方向：

自动卡被重物砸。
↓
进入眩晕。

自动卡被尖锐物攻击。
↓
抖动。
↓
红色数字。
↓
生命 -1。

眩晕会随时间恢复。

眩晕中的自动卡不能移动、捕食、采集、交易或逃跑。

# 4. 果实与植物繁衍

果实不是永久物资。

如果果实留在原地，没有被收集或吃掉。

下雨后，它可能成为下一代植物。

当前 MVP：

浆果留在地上。
↓
下雨。
↓
浆果变成灌木丛。

后续可以扩展：

- 种子
- 树种
- 草籽
- 水分/季节影响发芽

# 5. 玩家卡任务补全

游戏环境是围绕人物卡展开的。

如果人物卡对不齐，生态问题会被误判。

玩家卡至少需要参与：

- 收集食物
- 搬运到营地
- 有工具时屠宰尸体
- 没工具时表达缺少条件
- 把肉、浆果、木材、石头等放到营地附近

本轮仍不新增工具配方捷径。

玩家卡只使用既有关系。

如果缺少小刀，它会表达条件。

如果已经携带小刀或斧头，它会处理尸体。

《狼窝初始化、人物卡工具链与完整优先级 v2.16》

# 1. 本轮测试暴露三个根本问题

1. 狼没有狼窝时原地不动，筑巢逻辑只在吃饱后触发
2. 人物卡无法自主制造工具
3. 人物卡整体优先级链不完整

《篝火前提、需求驱动与河格规则 v2.17》

# 1. 篝火是世界的最大前提

篝火不是需求之一，篝火是其他需求得以成立的基础。

没有篝火：
- 狼不怕人
- 无法烹饪
- 无法形成营地
- 连安全都不成立

因此篝火是玩家卡的第一优先需求。

# 2. 篝火制作链（新增）

左键物理砸击，2 下触发：

木头 / 树枝  左键砸  碎石
↓
摩擦生火
↓
篝火（碎石消耗）

与石头链保持一致的节奏感（2 下触发，有抖动和红色数字）。

初始地图不再放置篝火。
玩家卡必须自主完成从材料到篝火的完整链条。

# 3. 河格规则

右键拖到河格的所有卡：

- 如有明确河格关系（如桶+河=一桶水）→ 执行关系，结果卡弹到旁边非河格
- 无关系 → 直接弹回，不落在河格上

唯一例外：未来某张卡右键叠河格改变了河格本质（如"桥"），
此时河格变成新格，无结果卡可弹。

# 4. 需求驱动架构

人物卡不再是命令式任务列表，而是需求数组驱动：

扫描所有需求
↓
找出最优先的未满足需求
↓
场地是否有满足条件？
↓
有 → 执行完整行动链（完全自主）
没有 → 表达缺少条件："XXX没有了，所以现在YY了"
↓
需求满足后继续循环
↓
所有需求满足 → 等待下一个需求出现

# 5. 人物卡需求优先级（确认版）

| 优先级 | 需求 | 说明 |
|---|---|---|
| 1 | 篝火 | 最大前提，解决驱狼/烹饪/营地 |
| 2 | 安全 | 有狼且无火庇护时立即逃跑 |
| 3 | 饱腹 | 饥饿100%后1.5游戏天死亡 |
| 4 | 居住 | 草棚保护篝火不被雨熄灭 |
| 5 | 基础工具 | 最低一件即停止，不重复制造 |

# 6. 饱腹需求的行为分支

每次饥饿触发时随机决定：

50% 浆果路线：
↓
采集灌木 → 得浆果 → 有营地则搬到营地 → 吃

50% 肉食路线：
↓
需要工具 → 无工具则制作 → 打猎 → 得肉 → 篝火烹饪 → 熟肉 → 吃

# 7. 饥饿死亡机制

饥饿值达到 100%
↓
开始倒计时（1.5 游戏天）
↓
倒计时归零 → 玩家卡死亡
↓
日志显示："玩家卡因长期饥饿倒下"

进食可重置倒计时。

# 8. 工具最低维护原则

携带至少 1 件工具即满足工具需求，停止制造。
工具消耗后（未来版本）或丢失后重新触发制造。
不无限制造同类工具。

# 9. 本轮验收目标

- 游戏开始后无篝火，人物卡主动制作篝火
- 木头砸碎石 2 下 → 篝火出现，有节奏感
- 有篝火后狼才开始惧怕
- 右键任意卡到河格，结果弹到旁边非河格
- 饥饿死亡倒计时可见
- 饱腹路线随机（浆果 or 肉食）

《狼窝初始化、人物卡工具链与完整优先级 v2.16》

# 1. 本轮测试暴露出三个根本问题

1. 狼没有狼窝时原地不动，因为筑巢逻辑只在"吃饱"后触发
2. 人物卡无法自主制造工具，只会等玩家手动提供
3. 人物卡整体优先级链还不完整，不够像"世界中的人"

# 2. 狼的初始化修复

狼的行为优先级调整为：

无狼窝且饥饿值 < 65
↓
先筑巢（找干草/草皮 → 搬到边缘 → 形成狼窝）

有狼窝，或饥饿值 ≥ 65
↓
捕猎

初始地图同步增加干草，保证狼一开始就有筑巢材料可用。

# 3. 人物卡工具制造链

## 核心原则

人物卡制造工具必须走与玩家上帝视角完全相同的关系链。
不走捷径，不凭空生成工具。

## 当前支持的最短工具链

石头 + 石头（砸 2 下）→ 碎石 ×2
↓
碎石（砸 2 下）→ 石制小刀

## 人物卡制造流程

1. 发现无工具且地图上无工具可拾取
2. 优先检查是否有碎石可直接塑形
3. 没有碎石则寻找 2 块石头
4. 走到石头旁边
5. 把一块石头移到另一块旁（世界中的"搬运"动作）
6. 执行砸击序列（等待 × 2 次），碎石产出，日志显示
7. 走向碎石
8. 执行塑形序列（等待 × 2 次），直接产出小刀
9. 走过去拾取工具

## 关键设计原则

上帝视角手动触发塑形 → 弹出对话框，玩家决定形状

人物卡自主触发塑形 → 根据自身需求直接执行，不弹框

这体现了：

上帝视角 = 玩家的灵感和干预
人物卡 = 有需求、有逻辑、会自己生活的化身

只要人物卡的需求是明确的，它会自己执行。
玩家不需要在中途做选择。

# 4. 人物卡完整优先级链

避险（发现狼且无火源屏障）
↓
吃饭（饥饿超过阈值 + 有可食物）
↓
执行进行中的工具制造任务（不因轻度饥饿中断）
↓
营地任务：
  有刀 + 发现羊尸体 → 屠宰
  无刀 + 发现羊尸体 → 触发工具制造链
↓
整理营地（搬食物/材料到营地附近）
↓
采集灌木 / 直接吃地上浆果
↓
缺材料时表达条件需求

# 5. 可食物范围扩展

人物卡可直接吃的食物：

- 蘑菇（原有）
- 熟肉（原有）
- 浆果（新增）：地上的浆果卡可被人物卡直接走过去吃

# 6. 本轮验收目标

玩家应能看到：

- 狼在游戏开始后优先筑巢，不再原地等待
- 人物卡在没有工具时，主动寻找石头制作小刀
- 制作过程可见：走向石头 → 砸击抖动 → 碎石出现 → 走向碎石 → 塑形 → 小刀出现

- 人物卡可以直接吃地上的浆果

《全局共性重构与需求内核 v2.25》

# 1. 本轮目标

此前多轮修复以“局部补丁”为主，已能止血，但会导致：

- 不同函数各自判断需求，出现重复制造或站桩
- AI 与手动交互在配方判定上不一致
- 有现成资源时仍重复新建
- 高优先级需求与低优先级任务相互覆盖

本轮目标不是再加新配方，而是统一“行为底盘”。

# 2. 唯一需求评估入口

人物卡所有行动必须先经过同一个需求评估内核：

evaluateNeedState()
↓
输出：唯一主需求 + 当前目标 + 执行计划

禁止在多个行为函数中直接触发行动。

需求链仍保持：

篝火 → 安全 → 饱腹 → 居住 → 工具 → 经营 → 无所事事扫描

并支持高优先级强制抢占低优先级。

# 3. 统一任务状态机

所有任务统一为：

plan
↓
move
↓
pickup
↓
move
↓
drop
↓
act
↓
done / fail

不允许某些任务绕过状态机直接生效。

这样可统一处理：

- 中断恢复
- 紧急抢占
- 失败回退
- 携带物落地

# 4. 统一配方引擎

配方校验必须唯一来源：

Recipe Engine（关系查询 + 可执行判定）

手动拖拽、AI自动、脚本触发都走同一个配方判断。

流程：

先验配方
↓
配方成立才执行
↓
失败则进入纠偏路径

这避免：

- 手动能合成但AI不能
- AI使用错误材料反复尝试
- 左右键语义冲突

# 5. 资源复用优先

所有需求先检查“已有可用成果”，再决定是否新建。

例如：

已有草棚但不在火域旁
↓
先搬棚
↓
仍不足时再新建草棚

已有工具可取
↓
先取工具
↓
缺失时再制作

这条是人物卡从“施工队”走向“决策者”的关键。

# 6. 统一放行门槛

经营需求只能在生存稳定后放行：

- 饱腹值 < 50
- 营地有至少2种食物
- 存在篝火
- 连续稳定 5 秒

放行条件失败则立即回到生存需求，不继续经营动作。

# 7. 统一失败记忆

人物卡应记录“失败原因短记忆”，避免下一帧重做同错动作。

典型失败：

- 目标材料被占用
- 配方不成立
- 路径不可达
- 目标已消失

失败后：

短冷却
↓
重评需求
↓
选择备用行动

这条用于消除“反复制造 / 反复拾放 / 原地循环”。

# 8. 本轮验收

玩家应能看到：

- 人物卡不会再长期重复同一错误动作
- 有现成资源时优先复用，不盲目新建
- 配方错误时自动纠偏，而不是死循环
- 高优先级需求能稳定抢占低优先级任务
- 经营只在稳定生存后出现

生活稳定条件：

- 有 fire
- 有 camp
- 饥饿 < 50
- 4 格内无未防火狼威胁

# 5. 蘑菇农 mushroomFarmer

## 5.1 数量与生成

- 全局最多 1 名 mushroomFarmer
- 条件：已有 mushroomGreenhouse，且地图上无蘑菇农
- 蘑菇棚存在后累计 2 游戏天 → 从地图边缘生成 seeking 状态农民
- 玩家拒绝招募：3 游戏天内不再生成

## 5.2 招募

seeking 状态：

- 移向蘑菇棚或 camp
- 玩家卡相邻 → 弹出招募面板（EventBus.recruit_prompt_requested）
- 接纳：join_state = employed，绑定 post_greenhouse_id
- 拒绝：join_state = leaving，离开地图

招募时不检查木头；接纳后由管理需求驱动玩家补木头。

## 5.3 雇佣后职责

- 蘑菇棚全套运维：采 mushroomWood、做 wetWood、打水、缺 bucket 时 spawn 近棚

| 原因 | 条件 |
|------|------|
| wood | 棚旁 1 格内 wood/twig 总量 < 2 |
| meat | 距上次 last_meat_day ≥ 2 游戏天 |
| hut | 棚旁 1 格内无「靠火第二草棚」（非主 hut） |
| protect | 狼距棚 ≤ 4 格 |
| feed | 农民 hunger ≥ PLAYER_HUNGER_NEED |

第二草棚：须在 mushroomGreenhouse 相邻 1 格，且该格 near_fire。

## 5.5 避险与死亡

- 狼距棚 ≤ 4：农民避险，向 fire 或玩家移动
- 饥饿 100% 持续 PLAYER_STARVE_DEATH_SECONDS → 移除卡，可再次按生成规则出现

《Capability / Action 分类法 v2.32》
日期：2026-05-27

# 0. 本节定位

本节记录世界规则标签机的下一层设计：能力 Capability 与行为 Action。

目标不是增加抽象复杂度，而是把后续越来越多的卡牌、生态、经营、AI 行为收束到稳定结构中，让创作有据可依，迭代成本逐步下降。

详细文档：`docs/design/capability-action-taxonomy-v0.1.md`

# 1. 核心原则

卡牌不应是一堆独立代码分支。

一张卡由四层封装：

- 静态标签：这张卡是什么
- 运行时状态：这张卡现在怎样
- 能力：这张卡原则上能做什么
- 行为规则：在什么条件下，对什么目标，执行什么行为，得到什么结果

标签只是入口，不是全部。

# 2. 三层分工

- 标签 Tag：描述稳定身份和属性，例如 `animal`、`predator`、`diet.meat`、`camp.storable`
- 能力 Capability：描述一张卡能进入哪个行为系统，例如 `capability.move`、`capability.hunt`、`capability.care_child`
- 行为 Action：描述实际动作模板，例如 `action.follow`、`action.hunt`、`action.feed`、`action.return_home`

原则：

- 标签回答“是什么”
- 能力回答“能进入哪个系统”
- 行为回答“现在做什么”
- 任务脚本只负责执行已被规则选中的行为，不应到处重新解释卡牌身份

# 3. 世界规则与非动物卡牌

动物只是样例。世界规则标签机必须覆盖所有卡牌，包括结构、场域、资源源、经营单位、环境锚点。

新增概念：

- `domain`：一张卡对周围空间建立的规则影响，例如营地域、狼群巢域、蘑菇生产域、资源源域
- `bond`：卡牌之间的稳定结构关系，例如篝火-草棚、篝火-桌子、草棚-蘑菇棚、狼窝-狼群
- `service`：结构或场域提供的服务，例如睡眠、庇护、交易、烹饪、收纳
- `produce`：资源源或生产结构提供产出，例如森林产树枝、蘑菇棚产蘑菇
- `transform`：输入被场域或结构转化，例如生肉在火边变熟肉、木材在火边变炭

样例：

- 篝火：`capability.define_domain`、`capability.protect`、`capability.transform_input`
- 草棚：`capability.bond_to_domain`、`capability.provide_service`
- 桌子：`capability.provide_service`、`capability.attract_actor`、`capability.trade`
- 蘑菇棚：`capability.define_domain`、`capability.produce_resource`
- 狼窝：`capability.define_domain`、`capability.support_reproduce`、`capability.provide_service`
- 森林：`capability.produce_resource`、`capability.regenerate`、`capability.respond_to_tool`
- 水源：`capability.produce_resource`、`capability.accept_container`

# 4. 捕猎与抚养样板

捕猎不应是狼专属，也不应是玩家专属。

`action.hunt` 应由 actor 的能力、食性、需求、目标状态共同决定：

- 狼：不使用工具，优先带肉回窝，可为幼崽捕食
- 玩家：可使用工具，可烹饪，可按营地储备决定是否继续狩猎
- 未来杂食动物：可在肉食和植物食物规则中择优

抚养也不应是幼狼专属逻辑。

`action.care_child` 可覆盖：

- 喂养
- 守护
- 带回家
- 无家时跟随成年个体
- 有家时走向家而不是瞬移

# 5. 迁移原则

后续不做一次性大重写。

推荐顺序：

1. `card-rule-audit.md` 已增加 `capabilities` 字段
2. 现有 50 张卡已补首批能力标记
3. 下一步同时覆盖生物与非生物：`hunt + care_child` 和 `camp domain + fire bond` 都应进入候选样板
4. 每次迁移都必须说明减少了哪些硬编码
5. 每次迁移都必须保留 L0-L3 验收

# 6. 验收

- 审计导出：通过，`card-rule-audit.md` 已生成 capabilities 列
- unit：310 assertions 通过
- 本节不改运行时玩法。

《Rule Dimensions / 标签维度约束 v2.36》
日期：2026-05-27

# 0. 本节定位

本节把“标签之上要有维度”落成工程约束。维度是世界语法，用来约束标签、能力、动作、域、羁绊、服务等规则词汇。

详细文档：`docs/design/rule-dimensions-v0.1.md`

# 1. 六个维度

- `identity`：卡牌稳定身份，例如 being、animal、structure、tool
- `material_form`：材料、形态、物理性质，例如 material.lumber、sharp、container.water
- `capability`：能进入哪个规则系统，例如 capability.hunt、capability.provide_service
- `relation_domain`：如何连接世界结构，例如 domain.camp、bond.fire、service.sleep
- `action`：当前可执行动作模板，例如 action.hunt、action.feed、action.follow
- `rule_modifier`：规则修饰、经济/生命周期/临时倾向，例如 commodity、food.raw、organize.locked

# 2. 工程约束

- `CardRuleAudit.dimension_for_token()` 负责给规则词汇归维度
- L0 检查 `CardDB` 标签、`CARD_CAPABILITIES`、首批运行时 `domain/bond/service/action` 全部能归入维度
- 新增自动事实表：`docs/design/generated/rule-dimension-facts.md`
- `card-rule-audit.md` 引用维度事实表

# 3. 当前边界

本轮没有立刻加入 `preference.*`、`reach.*`、`requires.*`、`risk.*`，因为这些应该随具体迁移案例进入系统：

- 猎物偏好迁移时再引入 `preference.rabbit`
- 武器距离迁移时再引入 `reach.1`
- 专名输入迁移时再引入 `requires.twig`
- 风险/禁忌迁移时再引入 `risk.near_fire`

# 4. 验收

- 审计导出：PASS，已生成 `rule-dimension-facts.md`
- L0 unit：PASS，345 assertions

《Hunt Target / 行为目标选择收口 v2.37》
日期：2026-05-27

# 0. 本节定位

本节是行为结构四段式中的 target 阶段样板。目标不是重写完整捕猎 AI，而是把“对谁做”从玩家 / 狼脚本中收口到 `WorldRules`。

详细文档：`docs/design/rule-dimensions-v0.1.md`

# 1. 已落地的规则查询

- `WorldRules.best_hunt_target(actor)`：返回 actor 当前最佳捕猎目标
- `WorldRules.hunt_target_visible_to(actor, target)`：判断目标是否对捕猎者可见
- `WorldRules.hunt_target_score(actor, target)`：集中表达目标排序和偏好
- `WorldRules.is_hunt_target_for(actor, target)`：集中表达目标合法性

# 2. 行为四段式映射

- `availability`：玩家是否需要肉、是否有武器；狼是否确实需要捕猎。本轮保留旧入口。
- `target`：已收口到 `WorldRules.best_hunt_target`。
- `execution`：走近、攻击、取肉仍由玩家任务 / 狼行为执行。
- `recovery`：手上有肉回窝、手持材料恢复任务等后续再迁移。

# 3. 卡牌状态修正

`CardBase` 补齐 `hiddenInGrass` 字段，使兔子藏草状态成为规则层可读的真实卡牌状态。狼在远距离不会把草中兔子当作可见猎物。

# 4. 设计边界

- 玩家和狼都通过 `capability.hunt` 进入捕猎结构。
- 玩家不继承狼，狼也不继承玩家；二者共同消费 target 查询。
- 狼偏好兔子仍暂时由 `hunt_target_score` 表达，后续可迁移为 `preference.rabbit`。
- 玩家本轮仍不自动猎羊羔，保持既有设计边界。

# 0. 本节定位

本节是行为结构四段式中 target / recovery 边界的第二个运行时样板。目标是把“哪个 home 属于这个 actor”从狼专名查找收口到 `WorldRules`。

详细文档：`docs/design/rule-dimensions-v0.1.md`

# 1. 已落地的规则查询

- `WorldRules.home_for_actor(actor)`：返回 actor 当前应返回的 home

- `availability`：actor 必须具备 `capability.return_home`

- `recovery`：`homeDenId` 绑定优先于最近狼窝，避免多狼窝场景误选近窝

# 3. 设计边界

- 成年狼和幼狼都通过 `capability.return_home` 进入回家结构。
- 狼窝通过 `domain.wolf_pack` 成为 home 候选。
- 本轮不重写筑窝、喂幼狼、叼肉清路，只替换 home 目标查询。
