# v3.0-A 四倍河谷生态 + 桃源三性格远观

> 状态：A1 ✅ provisional stable · **A2 ✅**（鹿生态链）· A3 待  
> 类型：规模 + 新卡 + 新规则  
> 长期主轴：完整自洽世界 / 完整桃源人部落  
> 本轮密度：4 倍面积河谷 + 6 张新卡 + 旅人封存 + 桌子饮食服务
> 工程交付：**必须按 A1 → A2 → A3 三次合入，每片单独验收**

## 1. 策划案

### 1.1 本轮设计目标

本轮不是单纯“把地图变大”，而是建立河谷世界语法的第一版。

当前地图仍接近测试棋盘：卡牌可以互动，但空间本身缺少天然属性，生态链也偏短。v3.0-A 要把世界推进到：

```text
空格天然属性
+ 卡牌占用 / 覆盖
+ 卡牌辐射的域
+ 自动卡读取世界规则并行动
= 一段能自运行、能反馈玩家行为的河谷边缘
```

玩家应感觉到自己不是在空棋盘上生存，而是在桃花源边缘的一段河谷里落脚。河流、林缘、草地、山岩和桃源边界都应成为世界事实。桃源人先作为远观者出现，有不同性格倾向，但不帮工、不交易、不成为员工池。

### 1.1a 工程切片原则

本文件是 v3.0-A 总案，不代表一次性合入。Cursor 必须拆成三片：

| 子切片 | 内容 | 验收 |
|---|---|---|
| **v3.0-A1** | terrain 数据源统一 + `world_rules_terrain` + 36×24 河谷地图 + 初始生存区保证 + terrain trace | L0 + terrain trace + L2b 1× |
| **v3.0-A2** | 鹿生态链：`deer` / `deerCorpse` / `deerMeat` + 狼/玩家猎鹿 + 烹饪链 | L0 + L1.5 鹿三卡 + 鹿专项 trace + L2b 1× |
| **v3.0-A3** | 桃源三性格远观 + 旅人 live 封存 + 桌子 dining + 轻量 bubble 里程碑 | L0 + L1.5 桃源三卡 + 社会远观 trace + L2b 1×；若改主链明显则 L2b 5× |

每片完成后都要更新 `docs/CODEX_HANDOFF.md` §12。A1 未通过前，不进入 A2；A2 未通过前，不进入 A3。

### 1.2 本轮验收句

> 地图明显像一段河谷，动物不再只有羊兔狼三件套；远处有三类桃源人在看我，他们有不同态度，但还不是我的人。玩家主链仍能稳定活下来并建成蘑菇棚。

### 1.3 世界四层语法

#### A. 空格层：没有牌的格子也有天然属性

空格不是卡牌，不进 `CardDB`。它是地图底层事实，由 `WorldRules` 查询。

| 空格属性 | 含义 | 本轮规则作用 |
|---|---|---|
| `terrain.river` | 河流 | 水源母体；可取水；普通生物不可站立；未来鱼 / 渡口 / 外部入口 |
| `terrain.riverbank` | 河岸 | 草木恢复更快；食草动物更常活动 |
| `terrain.grassland` | 草地 | 羊、兔、鹿的主要活动区 |
| `terrain.forest_edge` | 林缘 | 树、灌木、树枝、木材来源 |
| `terrain.stony_slope` | 山岩坡 | 石头、矿物、工具链起点 |
| `terrain.taoyuan_edge` | 桃源边界 | 桃源人远观、避让、社会边界 |
| `terrain.wildland` | 荒地 / 普通野地 | 默认自然地面，作为其他区域的过渡 |

要求：地图至少扩大到当前面积约 4 倍。建议长宽各约 2 倍，而不是线性 4 倍。

#### B. 卡牌层：卡牌仍是一切实体

卡牌落在空格上，并继承或响应空格属性。

示例：

- `tree` 更适合出现在 `terrain.forest_edge`。
- `grass` 更容易出现在 / 恢复于 `terrain.riverbank` 和 `terrain.grassland`。
- `sheep`、`rabbit`、`deer` 偏向 `terrain.grassland` / `terrain.riverbank`。
- `wolfDen` 应远离 `terrain.taoyuan_edge` 和玩家初始营地。
- `fire` 不改变地形，但辐射营地域 / 安全域。
- `mushroomGreenhouse` 建立蘑菇生产域。

#### C. 域层：卡牌或地形对周围产生影响

域不是装饰圈，而是影响规则判断。

| 域 | 来源 | 影响 |
|---|---|---|
| `domain.water_source` | `terrain.river` | 水桶可取水，未来动物饮水 |
| `domain.camp` | `fire` | 玩家营地、收纳、睡眠、饮食服务基础 |
| `domain.fire_safety` | `fire` | 抑制狼威胁，提供夜间安全 |
| `domain.greenhouse` | `mushroomGreenhouse` | 湿木、菇木、蘑菇生产 |
| `domain.predator_home` | `wolfDen` | 狼回窝、幼狼跟随、肉储备 |
| `domain.social_boundary` | `terrain.taoyuan_edge` + 桃源人 | 桃源人保持距离，玩家不能一开始直接进入社会内部 |

本轮不要求可视化所有域，但规则层必须能查询。

#### D. 自动卡层：有智能的卡读取世界规则

普通卡承载资源、结构、域或状态；自动卡读取世界规则并行动。

本轮新增自动卡不靠单独任务脚本硬跑，而应读取：

```text
我是谁的标签？
我在哪种 terrain？
我处在哪些 domain？
附近有哪些可交互卡？
当前世界规则允许我做什么？
```

## 2. 本轮新增 6 张卡

### 2.1 鹿生态链

本轮只新增一条完整生态位：鹿。不要同时扩狐狸、野猪、鱼等多条动物线。

#### 1. `deer` 鹿

定位：大型野生食草动物。比兔子更有肉，比羊更野，不属于玩家资产。

建议：

- `factory_lane`: `actor_loop`
- 标签：`being, animal, herbivore, wildPrey, largePrey`
- 能力：`capability.move`, `capability.forage`, `capability.flee`, `capability.be_hunted`
- 活动区：`terrain.grassland` / `terrain.riverbank` / `terrain.forest_edge`
- 避让：`domain.camp`、火域、桃源边界、狼
- AI 可达性：生态行为 / 玩家任务 / 手动交互

规则：

- 鹿会吃草或灌木嫩枝。第一版可复用羊 / 兔的觅食逻辑，不做复杂啃树。
- 狼饥饿时可捕猎鹿。
- 玩家有合适武器时可狩猎鹿。
- 鹿受惊时逃向远离火域、狼、玩家的草地或林缘。

#### 2. `deerCorpse` 鹿尸体

定位：鹿死亡后的中间态，进入食物链 / 清理链 / 腐解链。

建议：

- `factory_lane`: `structure_service`
- 标签：`corpse, organic, organize.locked`
- 能力：`capability.be_butchered`, `capability.sanitation_target`
- AI 可达性：玩家任务 / 生态 / 手动交互

规则：

- 可被玩家屠宰。
- 可被生态清理或后续腐解系统消费。
- 寻路阻挡，不能被普通收纳。

#### 3. `deerMeat` 鹿肉

定位：生肉，进入现有烹饪链。第一版不要新增“烤鹿肉”卡，继续产出通用 `cookmeat`。

建议：

- `factory_lane`: `production_node`
- 标签：`consumable, food, food.raw, raw, camp.storable`
- 能力：`capability.be_carried`, `capability.be_stored`, `capability.be_consumed`, `capability.be_cooked`, `capability.be_traded`
- AI 可达性：玩家任务 / 手动交互 / 收纳

规则：

- `deerMeat + fire -> cookmeat`。
- 可被营地收纳。
- 可作为狼肉储备候选，但狼优先不抢营地内玩家食物；本轮若复杂，可先不让狼拿营地鹿肉。

### 2.2 桃源三性格远观

三种桃源人都是“社会远观原型”，不是员工、不是旅人、不是交易 NPC。

共同规则：

- 不进入玩家营地中心。
- 不自动帮工。
- 不交易。
- 不接任务。
- 保持与玩家卡的社交距离。
- 观察玩家关键里程碑：生火、草棚、桌子、蘑菇棚、狩猎鹿。
- 被玩家靠近时，根据性格后退、停留或绕开。

#### 4. `taoyuanElder` 桃源长者

定位：守边界、谨慎、重传统。代表桃源社会边界。

建议：

- `factory_lane`: `actor_loop`
- 标签：`being, human, taoyuan, autonomous, observer, elder`
- 能力：`capability.move`, `capability.observe`, `capability.flee`, `capability.social_boundary`
- AI 可达性：生态 / 社会行为

行为：

- 距离玩家最近安全距离最大。
- 更少靠近营地。
- 对火、棚、猎鹿有观察，但更常表达警惕。

bubble 示例：

```text
“外来人会生火。”
“别太近。”
“他在取这片林子的东西。”
```

#### 5. `taoyuanForager` 桃源采集者

定位：懂本地生态、河岸、林地。代表桃源人的地方知识。

建议：

- `factory_lane`: `actor_loop`
- 标签：`being, human, taoyuan, autonomous, observer, forager`
- 能力：`capability.move`, `capability.observe`, `capability.forage`, `capability.social_boundary`
- AI 可达性：生态 / 社会行为

行为：

- 更常出现在河岸、林缘、草地边缘。
- 关注玩家砍树、用水、建菇棚、捕猎鹿。
- 不直接采集给玩家，不加入营地。

bubble 示例：

```text
“他知道从河边取水。”
“那棚子像是在养东西。”
“鹿被赶到林边去了。”
```

#### 6. `taoyuanYouth` 桃源青年

定位：好奇、靠近倾向更强，是未来最早被新商品 / 新服务影响的人群原型。

建议：

- `factory_lane`: `actor_loop`
- 标签：`being, human, taoyuan, autonomous, observer, youth`
- 能力：`capability.move`, `capability.observe`, `capability.social_boundary`, `capability.attracted_by_novelty`
- AI 可达性：生态 / 社会行为

行为：

- 比长者更接近玩家营地，但仍保持距离。
- 对桌子、熟肉、蘑菇棚、鹿肉等“新组合”更有反应。
- 玩家靠近时先后退，不直接对话。

bubble 示例：

```text
“他把肉放在火边。”
“那张桌子是做什么的？”
“我想再看看。”
```

## 3. 旅人封存与桌子降级

### 3.1 旅人 live 封存

`traveler` 暂时作为备用卡封存。

当前规则：

- 不自然生成旅人。
- 不触发旅人交易循环。
- 不作为当前商业链验收目标。
- 保留 CardDB、标签、审计记录。
- 保留 commerce 单元测试；测试 fixture 可以手动 spawn traveler。
- `traveler` 可继续保留在 factory smoke 默认列表，作为“卡定义仍有效”的守卫。
- 审计 risk 标注：商业生态不足，旅人主循环待后端生产 / 社会反应成熟后解封。

设计理由：

商业生态尚不充分。若此时让旅人活跃，会迫使代码为了“旅人来问、吃、走”到处补条件，形成假商业系统。

### 3.2 桌子改为饮食服务建筑

`table` 暂时不作为经营单位主舞台，而是营地饮食建筑。

当前身份：

```text
table = service.dining
```

规则：

- 桌子在 `domain.camp` 内才提供饮食服务。
- 玩家卡在桌旁 1 格进食时，饥饿恢复更多。
- 桌子不吸引旅人。
- 桌子不触发交易 UI。
- 未来可升级：

```text
table v1：吃饭服务
table v2：摆放商品
table v3：公开交易
table v4：契约 / 岗位 / 账目
```

建议第一版数值：

- 桌旁进食恢复量乘以 `1.3`。
- 若现有饥饿系统更适合固定值，可用固定额外恢复，但必须写入 `WorldRules` 查询。

## 4. 地图与生态规则

### 4.1 地图规模

目标：面积约 4 倍。建议长宽各约 2 倍。

必须保证：

- 初始可见区域仍有最小生存前提：树枝 / 石头 / 草 / 水源可达。
- 地图扩大不能导致玩家前 1 日长期乱逛。
- 蘑菇棚主链仍可建成。

### 4.2 区域分布

建议结构：

```text
上游 / 林缘 / 山岩坡
        ↓
河流贯穿地图
        ↓
河岸 / 草地 / 玩家初始营地
        ↓
桃源边界在地图一侧或远端
```

本轮只需要“可读的河谷”，不要求复杂地形生成器。

### 4.3 生态分布

- 羊 / 兔 / 鹿偏向草地、河岸。
- 狼窝偏向远离玩家初始营地和桃源边界的位置。
- 鹿比兔更容易被狼关注，但狼食物足够时不重复猎杀。
- 鹿不进入桃源边界和玩家火域中心。
- 河岸提高草恢复概率，但本轮不做完整水文。

## 5. Cursor 执行单

### 5.1 总实现顺序

建议 Cursor 按顺序实现，不要并行大改。下列顺序必须按 A1 / A2 / A3 分三次合入。

#### v3.0-A1：河谷地形基础

0. **统一现有 terrain 数据源**：先让 `TerrainManager` 与 `Pathfinding` 共用同一套 cell 分类，或二者都走 `WorldRules.terrain_tags_for_cell()` / 等价 facade。不得新增第三份河流 / 河岸公式。
1. 建立 terrain / zone 查询：地图格天然属性可被 `WorldRules` 查询。建议新增 `world_rules_terrain.gd`。
2. 地图从 `18×12` 扩到 `36×24`，并迁移所有硬编码 spawn / exit / 初始卡位置。不能只翻倍常量。
3. 初始可见区强制保证最小生存资源：树枝 / 石头 / 草 / 水源可达。
4. 新增 terrain 专项 trace。
5. 跑 L0 + terrain trace + L2b 1×。

#### v3.0-A2：鹿生态链

前置：A1 已通过。

1. 新增鹿生态链三卡：`deer`、`deerCorpse`、`deerMeat`。
2. 接入鹿的生态行为：觅食、逃跑、被狼 / 玩家捕猎、死亡产尸体、屠宰产鹿肉、鹿肉可烹饪。
3. 控制鹿数量和肉峰值，避免 `meat_peak` 爆炸。
4. 补鹿专项 trace。
5. 跑 L0 + L1.5 鹿三卡 + 鹿专项 trace + L2b 1×。

#### v3.0-A3：桃源远观 + 旅人封存 + 桌子 dining

前置：A2 已通过。

1. 新增桃源三性格卡：`taoyuanElder`、`taoyuanForager`、`taoyuanYouth`。
2. 接入远观 / 避让 / 简单 bubble，不做交易、不做雇佣。
3. 封存 `traveler` live 生成与交易主循环，但保留 CardDB、单元测试和 factory smoke。
4. 桌子改为 `service.dining`，桌旁进食加成；暂不承担商业入口。
5. 补社会远观 trace。
6. 跑 L0 + L1.5 桃源三卡 + 社会远观 trace + L2b 1×。若改动主链明显，最终再跑 L2b 5×。

#### 最终收尾

1. 刷新审计文档：`tools/export_card_rule_facts.ps1`。
2. 更新 `docs/design/CHANGELOG.md`。
3. 每个子切片都在 `docs/CODEX_HANDOFF.md` §12 追加短条。

### 5.3 必须走 WorldRules 的查询

不要在任务层散落坐标判断或卡名判断。建议新增或等价实现：

```gdscript
WorldRules.terrain_tags_for_cell(cell)
WorldRules.cell_has_terrain(cell, "terrain.riverbank")
WorldRules.zone_for_cell(cell)
WorldRules.is_water_source_cell(cell)
WorldRules.is_taoyuan_boundary_cell(cell)
WorldRules.actor_social_distance_min(actor)
WorldRules.taoyuan_should_avoid(actor, target_cell)
WorldRules.is_dining_service_provider(card)
WorldRules.food_restore_multiplier_for_context(actor, food, cell)
WorldRules.is_traveler_enabled()
```

如实际命名不同，可以调整，但必须保持 facade 查询语义集中。

注意：第一版不要做通用 `social_observer_reaction(actor, event)` facade，避免把 A3 拖成小型 IntentEngine。里程碑观察用轻量 `GameState` 标志位或现有 EventBus 的 2–3 个事件即可；bubble 文案每种性格 3–5 条即可。

### 5.4 卡牌工厂要求

对 6 张新卡全部执行，但按子切片分批：

```powershell
.\tools\run_card_factory_scaffold.ps1 <type> --tags=... --name=... --write-dir=generated/scaffold
```

合入后刷新：

```powershell
powershell -File tools/export_card_rule_facts.ps1
```

并确保：

- `factory_lane` 正确。
- 标签维度归类完整。
- `CardDB` relation / impact 引用完整。
- L1.5 可指定单卡或多卡 smoke。
- 新增 `capability.observe`、`capability.social_boundary`、`capability.attracted_by_novelty` 等能力前，必须同步 `CardRuleAudit` 和维度归类；若实现成本高，A3 第一版可少加 capability，只在行为逻辑里表达远观 / 避让。

### 5.5 禁止事项

本轮不要做：

- 4–6 倍完整世界，只做 4 倍面积河谷。
- 50 个桃源人。
- 完整部落文化系统。
- 雇佣 / 岗位 / 契约。
- 旅人交易主循环。
- 腊肉。
- 鱼、狐狸、野猪等第二条动物线。
- 完整生态容量模拟。
- 为了让鹿或桃源人跑通，在 player/world/ui/input 散落 `card_type ==`。
- 让桃源人进入营地帮忙。
- 让桌子继续承担商业交易入口。
