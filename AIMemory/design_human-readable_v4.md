# 方寸商国：桃花源记 — 设计规范（人读版）

> 状态：讨论完成，部分待机读化。2026-06-11。

---

## 抽象原则

1. **参考来源**：真实世界、科学研究。**抽象方式**：由我们游戏的结构决定创作逻辑。不是复制现实，是提取能产生有意义差异的特质封装为标签。

2. **区分本体特质 vs 环境结果**：
   - 卡本身固有的 → 标签写在本体上
   - 环境导致的 → 不写标签，是生态计算的结果
   - 例：兔幼崽高死亡率 → 不是 `mortality:high` 标签，是"小型+多猎手+无防御"在生态中的自然结果。兔的天生特质是 `prolific`（高繁殖），不是"容易死"。

---

## 铁律：禁止硬编码

所有行为必须通过标签+公理涌现。不能有任何 `if type_name == "wolf"` 式的物种分叉。行为差异来自**真实世界对象上能被抽象出来的有价值的典型特质**——封装为标签。架构不完善可以完善架构，但不能绕过架构硬写。

## 第一部分：世界是什么

### 1.1 基础：格子与卡

游戏只有两种东西：**格子**和**卡**。

- **格子**是放置卡的位置。格子本身也是一组封装的标签。
- **卡**是一组封装的标签。每张卡的本质不是它的名字，是它携带的标签集合。

### 1.2 卡的两大分类

| 分类 | 特征 | 例子 |
|---|---|---|
| **自动卡** | 有智能，自主决策 | 类人（玩家/村民）、动物（狼/羊/鱼） |
| **物体** | 不动，无自主行为 | 山、石、草、木、工具、肉 |

### 1.3 创建新卡的原理

放入任何标签组合就形成一张新卡。游戏本质是用**抽象逻辑的标签模拟真实世界的复杂系统**。标签是通行证——有通行证就能通过因果引擎的门禁，没有就被挡在外面。

### 1.4 标签设计的主客体思路 ★

- **主体的自愿行为** → 标签写在主体身上
- **客体的被动触发** → 标签写在客体身上

### 1.5 因果引擎

- 类比物理引擎——保证世界基本模拟不出荒谬
- 引擎规则固定，不随新卡增加而改变
- 只能挡住"没有通行证的"，对有通行证的毫无办法
- 标签创作完全自由

### 1.6 格子的标签

格子也是一组标签。卡的标签和格的标签可以产生关系（查标签组合规则表）。

### 1.7 藏 = 容纳

藏是暂时的容纳关系，和固定居住关系本质相同。详细规则见第八部分。

## 第二部分：交互系统

### 2.1 左键砸

- 一切攻击的本质就是砸
- 有 HP → 攻击：1砸=1HP；无 HP → 加工：2砸=出产品
- 不可连续蹭，拉开再碰才算第 2 下
- 红色圆圈显示"砸"→2→3……

### 2.2 右键叠

- 叠是改变本质。两种结果由目标卡标签决定：
  - 组合变新卡 / 加标签
- 不兼容时弹回
- 半透明幽灵移动，蓝色圆圈显示"叠"

### 2.3 操作判定

- 砸还是叠：操作者决定（左键/右键）
- 能不能砸/叠、砸/叠之后怎么变：目标卡的标签决定

## 第三部分：因果引擎

### 3.1 四条公理

| 公理 | 管什么 |
|---|---|
| compose | 能不能进入这个格 |
| traverse | 能不能跨介质移动 |
| perceive | 能不能感知到目标 |
| transform | 能量怎么转化 |

### 3.2 引擎的边界

公理只管"能不能"，不管"想不想"。动物行为在标签层。

### 4.1 两个独立维度

| | 可移动 | 不可移动 |
|---|---|---|
| **有智能** | 玩家、狼、羊、鱼 | 植物人（可思考不可动） |
| **无智能** | 风滚草 | 草皮、山、石头、腐殖土 |

## 第五部分：涌现哲学

### 5.1 核心

每张卡是标签的集合。行为由所有标签的互相关系涌现。整个游戏的运行是所有卡的标签在世界上互相碰撞涌现出来的。游戏玩的是"关系"。

### 5.2 与工程的关系

不需要中央"智能"引擎。行为分散在各标签里。差异在标签参数，不在代码分支。

## 第六部分：智能卡的驱动力系统

> 参考：Max-Neef 九需求、MOOGAL 八活动、Dwarf Fortress 特质/价值观/需求。

### 6.1 三层标签

| 层 | 标签格式 | 示例 |
|---|---|---|
| 特质 | `trait:curiosity(70)` | 0-100 数值，稳定但可变 |
| 价值观 | `value:knowledge` | 信仰什么 |
| 需求 | `need:eat(rate=2,curve=steep)` | 从特质×价值观自动派生 |

### 6.2 需求驱动行为

```
特质 + 价值观 → 需求强度 → 效用打分 → 选最高分 drive → 执行 → 需求重置
```

---

## 第七部分：决策引擎

### 7.1 效用打分（从 The Sims + IAUS 学习）

不写死优先级。需求紧迫度随时间动态变化。引擎每 tick：need 值衰减累积 → 所有 drive 同时打分 → 选最高分执行 → 满足后归零。

### 7.2 场论解释

标签 = 场中的荷。公理 + 需求（势能）= 卡牌在公理场中自动受力演化。行为是公理推导的结果。

### 7.3 自由度超越传统

需求本身也是标签，无限增加。标签互激。事件改变标签。标签组合产生新需求。

## 第八部分：藏=容纳

### 基本机制

藏 = 暂时容纳。与鸟巢在树上、水藻在水潭中完全相同。动物钻入 → `in_cover=true`，不占格、不渲染。覆盖物右上角紫圆白"藏"字。

### 感知规则

| 场景 | 结果 |
|---|---|
| 狼在远处 | 只感知草皮，不感知兔 |
| 狼在邻格+嗅觉灵敏 | 感知到兔 |
| 连续草丛 | 直接切换容纳，不暴露 |
| 不连续处 | 跑出→暴露→再进入 |

### 释放规则

- 草皮破坏 → 兔弹出到任意邻格
- 安全/饿了 → 兔主动出草

## 第九部分：世界维度

> 「当前」= 现阶段落地。「未来」= 生态稳定后。

### 9.1 温度与季节「当前」

- 季节波动，温度可局部改造（篝火）
- 冬天水潭结冰 → 冰卡覆盖 → 可通行/破坏/采集

### 9.2 衰老「当前」

- 超最大寿命 → 自然死亡
- 老年标签：`trait:frail`、`trait:slow`、`trait:wise`

### 9.3 生态多样性「当前」

#### 生产者（按空间分层）

| 层 | 类型 | 条件 |
|---|---|---|
| 树冠 | 橡果、松果、浆果 | 需大树 |
| 地表 | 湿地草、旱地草、菌菇、蕨类 | 看土壤 |
| 地下 | 山药、根茎、块茎 | 需松/深土 |

#### 地块标签

| 类型 | 标签 | 分布 |
|---|---|---|
| 旱地 | `soil:dry, fertility:low` | 离水远 |
| 沃土 | `soil:rich, fertility:high` | 水潭周边 |
| 林地 | `soil:loose, shaded` | 大树周边 |
| 石地 | `soil:rocky, fertility:none` | 山脚 |
| 深土 | `soil:deep, fertility:high` | 少数 |
| 湿地 | `soil:wet, fertility:high` | 水潭+外圈 |

半随机分布，水潭和湿地逻辑绑定。

### 9.4 分解者与物质循环「当前」

- 分解者：腐殖土、陆虫、菌类、蚯蚓、粪虫
- 动物排便 → 粪虫分解 → 土壤肥力
- 完整链：食物→消化→排泄→分解→土壤→植物→食物
- 分解者可被互动、破坏、采集

### 9.5 腐败系统「当前」

| 阶段 | 标签 | 特征 |
|---|---|---|
| 新鲜 | `decay:fresh` | 正常 |
| 初期 | `decay:early` | 变色 |
| 中期 | `decay:advanced` | 软化 |
| 严重 | `decay:severe` | 濒崩 |

触发：死亡、被切断、潮湿、高温。速度受环境影响。最终→腐殖土。

### 9.6 疾病「当前」

必须因果触发：吃腐肉→中毒、受寒→发烧、咬伤→感染。疾病标签影响行为。

### 9.7 手牌系统「未来」

社会层初步落地后设计。右击叠变为手牌使用方式。

### 9.8 社会层「未来」

生态稳定后推进。交易、地位、交流、学习。
