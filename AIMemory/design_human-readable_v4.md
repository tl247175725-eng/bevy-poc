## 第一部分：世界是什么

### 1.1 基础：格子与卡

游戏只有两种东西：**格子**和**卡**。

- **格子**是放置卡的位置。格子本身也是一组封装的标签。
- **卡**是一组封装的标签。每张卡的本质不是它的名字，是它携带的标签集合。

> 例：狼 = 捕食者、肉食、群居…… 这些标签才是狼的本质。"狼"只是给玩家看的名字。

### 1.2 卡的两大分类

| 分类 | 特征 | 例子 |
|---|---|---|
| **自动卡** | 有智能，自主决策 | 类人（玩家/村民）、动物（狼/羊/鱼） |
| **物体** | 不动，无自主行为 | 山、石、草、木、工具、肉 |

### 1.3 创建新卡的原理

放入任何标签组合就形成一张新卡，甚至可以是恶魔。游戏的本质是用**抽象逻辑的标签模拟真实世界的复杂系统**。标签是通行证——有通行证就能通过因果引擎的门禁，没有就被挡在外面。

### 1.4 标签设计的主客体思路 ★

设计标签时区分**主体**和**客体**：

- **主体的自愿行为** → 标签写在主体身上
  - 例：兔子饿了自动从草里出来（`hide:exit_when_hungry` → 标签在兔子）
  - 例：兔子觉得安全了自动从草里出来（`hide:exit_when_safe` → 标签在兔子）

- **客体的被动触发** → 标签写在客体身上
  - 例：草皮被破坏 → 释放所有内容物（`cover:release_on_destroy` → 标签在草皮）
  - 例：沼泽让经过的卡陷入（`trap:engulf` → 标签在沼泽）

### 1.5 因果引擎

- 因果引擎类比物理引擎——保证世界基本模拟不出荒谬（狐狸不能在水里生活、鱼不能在地上行走）
- 引擎的规则是固定的，不随新卡增加而改变
- 引擎只能挡住"没有通行证的"，对有通行证的毫无办法
- 标签创作完全自由——恶魔卡只要有对应的标签就能做任何事

### 1.6 格子的标签

- 格子也是一组标签。上面没有卡时，格子的标签决定"这是什么"
- 上面有卡时，卡的标签和格的标签可能产生关系（如篝火放水格上 → 查标签组合规则表 → 变木炭）
- 关系规则按标签组合匹配，不是按卡名匹配
  - 例：`fire + wood + water` → 木炭（精确三个条件，不是有 fire 就行）

### 1.7 藏 = 容纳

藏是**暂时的容纳关系**，和固定的居住关系（狼窝、鸟巢）本质相同。

- **触发**：兔子在草皮格中且有威胁 → 执行藏
- **藏之后**：兔卡在地图上消失，彻底容纳在草皮内部。不占格。
- **查看**：点击草皮 → 右侧详情页有"兔子"词条 → 点击词条 → 看兔卡详情
- **释放条件**：
  - 主体（兔）自愿：安全了自动出来（标签在兔）、饿了自动觅食（标签在兔）
  - 客体（草）被动：草被破坏 → 所有内容物弹出（标签在草）

## 第二部分：交互系统

### 2.1 左键砸

砸是物理动作。一切攻击的本质就是砸。

- **判定攻击还是加工**：看被砸的卡有没有 HP。
  - 有 HP → 这是攻击：1砸 = 1HP
  - 无 HP → 这是加工：2砸 = 出产品
- **不可连续蹭**：拉开再碰才算第 2 下
- **视觉**：红色圆圈显示"砸"→2→3……

### 2.2 右键叠

叠是改变本质。

- **两种结果**：由**被叠卡的标签**决定走哪条路径
  - 组合变新卡：标签组合匹配规则表 → 原卡消失，生成新卡
  - 加标签：原卡保留，标签列表增加
- **不兼容时弹回**：比如对石头叠"愤怒"——石头没有能接受愤怒的标签 → 弹回手卡
- **标签叠加有没有上限**：工程上没有（Vec<String>无限 push）。设计上有——如"愤怒+愤怒=暴怒"，"愤怒+冷静=抵消"。这些是具体组合规则，未来持续梳理
- **视觉**：半透明幽灵移动，蓝色圆圈显示"叠"

### 2.3 操作判定

- 砸还是叠——**操作者决定**（左键/右键）
- 能不能砸/叠、砸/叠之后怎么变——**目标卡的标签决定**

## 第三部分：因果引擎

### 3.1 定位

因果引擎类比物理引擎——模拟真实世界基本规律。保证不出现荒谬（狐狸水里生活、鱼地上行走）。

引擎规则固定，不随新卡增加而改变。只能挡住"没通行证的"，对有通行证的毫无办法。

### 3.2 四条公理

| 公理 | 管什么 |
|---|---|
| compose | 能不能进入这个格（空间承载） |
| traverse | 能不能跨介质移动（陆地→水域） |
| perceive | 能不能感知到目标（跨介质、范围） |
| transform | 能量怎么转化（吃、杀、采集） |

### 3.3 引擎的边界

动物具体的行为（怎么追、怎么逃、什么时候躲）不在公理层——是设计层和标签层的事。公理只管"能不能"，不管"想不想"。

## 第四部分：卡的类型

### 4.1 两个独立维度

所有卡按两个维度交叉分类：

| | 可移动 | 不可移动 |
|---|---|---|
| **有智能** | 玩家、狼、羊、鱼 | （植物人——可思考不可动） |
| **无智能** | 风滚草（随风动但无决策） | 草皮、山、石头、腐殖土 |

- **可移动** = 能从一格走到另一格（位置变化）
- **有智能** = 有驱动行为的逻辑（seek/flee/forage 等）
- 原地变化（腐殖土长草）≠ 移动

## 第五部分：涌现哲学

### 5.1 核心

每张卡是标签的集合。一张卡最终呈现什么行为，是由它所有标签的互相关系**涌现**出来的。

整个游戏的运行，是所有卡的标签在世界上互相碰撞所**涌现**出来的。

游戏玩的是"**关系**"——像矮人要塞那样。

### 5.2 与工程的关系

不需要一个叫"智能"的中央引擎来判断每张卡做什么。行为分散在各个标签里。

- `drive:seek` 让人和狼都能寻找目标，但 target 标签不同
- `craft:tool` 让人能做复杂工具，`craft:simple` 让猴子只能用树枝
- 差异在标签的参数，不在代码分支

## 第六部分：智能卡的驱动力系统

> 参考：Max-Neef 九需求、MOOGAL 八活动、Dwarf Fortress 特质/价值观/需求三层。

### 6.1 智能卡的本质

智能卡也是标签的集合。它的行为不是行为树，是**标签驱动的涌现**。

### 6.2 三层标签（从 DF 学习）

| 层 | 说明 | 例子 | DF对应 |
|---|---|---|---|
| **特质层** | 0-100数字，稳定但可变 | `trait:curiosity(75)`、`trait:anger(20)` | Facets |
| **价值观层** | 信仰什么、看重什么 | `value:knowledge`、`value:cooperation` | Values |
| **需求层** | 从特质×价值观自动派生 | 好奇心高+看重知识 → `need:explore` | Needs |

### 6.3 需求对标（Max-Neef + MOOGAL）

| Max-Neef 需求 | 游戏中标签方向 | MOOGAL 活动映射 |
|---|---|---|
| **生存** | `need:food` `need:shelter` `need:health` | 食物供应、生理 |
| **保护** | `need:safety` `need:defense` | 环境维护 |
| **情感** | `need:bond` `need:care` | — |
| **理解** | `need:learn` `need:explore` | 教育、体验 |
| **参与** | `need:trade` `need:cooperate` | 组织 |
| **休闲** | `need:rest` `need:play` | 体验 |
| **创造** | `need:craft` `need:build` | 非食物采集、食物供应 |
| **身份** | `need:status` `need:role` | — |
| **自由** | `need:autonomy` `need:wander` | 交通 |

### 6.4 需求驱动行为（核心引擎）

```
特质(trait) + 价值观(value) → 需求强度(need:N)
需求强度变化 → 选择最高优先需求 → 转化为 drive
drive → 引擎执行 → 需求满足 → 需求值重置
```

例：
- `trait:curiosity(70)` + `value:knowledge` → `need:explore(7)` 高
- 同时 `need:food(9)` 也高（饿了）→ food 优先级更高 → seek:foodSource
- 吃饱后 food→0 → explore 成为最高 → wander 到未探索区域

### 6.5 标签自由度

标签不受限制。短期（`anger:临时(3tick)`）、长期（`irritability:70`）、学习获得（`ninja_body:50`）都可以。涌现深度不设上限。

## 第六部分（接）：玩家卡的设计路径

### 当前状态

玩家卡的基础生存标签未完整激活。已激活的有觅食、逃跑。缺失的是制造工具、建草棚、篝火、社会交互等。

- 例：`一日三餐` 标签 → 每天必须吃 3 次，否则负面效果
- 例：`制作工具` 标签 → 条件满足时优先做工具，而不是用拳头砸羊
- 例：`好奇心` 标签 → 看到新东西主动走过去交互 → 可能触发制作工具→……

一个人类的日常生活和发展，就是由这些标签互相碰撞涌现出来的。

### 6.2 当前状态

玩家卡的基础生存标签未完整激活。已激活的有觅食、逃跑。缺失的是制造工具、建草棚、篝火、社会交互等。

### 6.3 研究任务

需要基于社会学、人类学、哲学的资料，设计一个人卡的基础标签集合。目的是涌现出日常生活和自主发展行为。

## 第七部分：决策引擎（已达成共识）

### 7.1 不是固定优先级

不写死 `priority:3`。需求紧迫度**随时间动态变化**。

### 7.2 效用打分（从 The Sims + IAUS 学习）

引擎每 tick 做的事：
1. need 值衰减累积（饥饿增长、恐惧随距离变化）
2. 所有可能的 drive 同时打分（need值 × 曲线 × 距离衰减）
3. 选最高分执行
4. 满足后该 need 归零

### 7.3 标签三层的落地方案

| 层 | 标签格式 | 示例 |
|---|---|---|
| 特质 | `trait:curiosity(70)` | 0-100 数值，稳定但可变 |
| 价值观 | `value:knowledge` | 信仰什么 |
| 需求 | `need:eat(rate=2,curve=steep)` | 从特质×价值观自动派生 |

### 7.4 场论解释

标签 = 场中的荷。公理（compose/perceive/transform）+ 需求（势能）= 卡牌在公理场中自动受力演化。行为是公理推导的结果，不是缝补的响应。

### 7.5 自由度超越传统

- 需求本身也是标签，无限增加
- 标签互激：环境标签 × 个人标签 → 新需求
- 事件改变标签：被狼追过 → trait:courage 下降
- 标签组合产生新需求：好奇心 + 忍者体质 + 日式建筑 → need:learn_ninjutsu
