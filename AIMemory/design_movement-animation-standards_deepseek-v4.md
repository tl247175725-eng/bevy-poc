# 移动动画设计标准

## 基准

以人类卡为基准：**0.25 秒/格**，参考 D20 RPG、Fire Emblem、Advance Wars 级别战术游戏的手感。

## 速度标尺

| 标签值 | 每格时长 | 视觉感受 | 示例 |
|---|---|---|---|
| `slow` | 0.35s | 慢悠悠 | 牛、水牛 |
| `normal` | 0.25s | 正常走路 | 人、村民 |
| `fast` | 0.18s | 较快 | 狼、鹿 |
| `very_fast` | 0.12s | 很快 | 兔 |
| `sprint` | 0.08s | 冲刺（追踪极限） | 猎豹冲刺 |

**原则**：速度参考人眼追踪上限，不是生物速度比。每格最短不低于 0.06s，否则变成闪现。

## 移动规则

1. **只能走上下左右**——不允许斜线移动。斜线拆为"先横后纵"两步 Sequential 动画。
2. **动画阻塞 tick**——所有移动动画播完才进下一个 tick。模拟层和表现层对齐。
3. **速度来自标签** `move_speed: normal`，引擎读取标签 → 查表 → 确定时长。标签不写具体秒数，写语义名。
4. **新卡默认 `normal`**——无 `move_speed` 标签时默认 0.25s/格。

## 将来可扩展的动画标签

```
move_speed: sprint        # 移动速度
move_easing: elastic_out   # 缓动曲线（默认 CubicOut）
move_trail: true           # 移动拖尾
teleport: true             # 跳过动画（瞬移，幽灵/魔法）
```
