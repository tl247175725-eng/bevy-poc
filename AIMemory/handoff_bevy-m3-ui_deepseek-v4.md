## M3 目标

把 Godot 的 UI 体验迁移到 Bevy。格网变 tile、卡可选中、信息面板、容纳列表、拖拽。不做精灵图——继续色块。让游戏从"看生态跑"变成"能交互"。

---

## 一、格网 Tile 化

### 1.1 当前状态

`grid_render.rs` 用 Bevy Sprite 画色块。每个格一个 sprite，每个卡一个 sprite。

### 1.2 改造

- 格子用 `bevy_ecs_tilemap` 或自研 tile 批次渲染（36×24 格，不用每个格一个独立 sprite）
- 卡牌 sprite 仍然独立——需要在格子上层绘制
- 格网背景色从 `terrain_colors.rs` 读取

### 1.3 相机缩放

- 加 `bevy_pancam` 或手动滚轮缩放
- 默认视口：36×24 格全显示
- 滚轮放大缩小
- 中键拖拽平移

---

## 二、卡牌选中 + 高亮

### 2.1 点击选中

- 鼠标左键点击格子 → 如果有卡在该格，选中卡
- 选中卡 → 绿色边框描边（`(50, 220, 80, 255)` 2px）
- 若格内有 `in_pool`/`in_tree`/`in_ground` 卡（隐藏卡）→ 优先选中地面可见卡，面板显示"容纳"入口

### 2.2 选中格（空地）

- 点击空格 → 显示格信息
- 格信息：地形类型、海拔、地下内容（如有）

## 三、信息面板

### 3.1 布局

右侧区域：固定宽度 ~250px

```
┌─────────────┬──────────────────┐
│   格网      │   信息面板        │
│  36×24      │                  │
│             │  【羊】           │
│             │  身份：动物/食草   │
│             │  能力：移动/觅食   │
│             │  状态：吃草       │
│             │  目标：草皮       │
│             │  HP：2           │
│             │  性别：公         │
│             │                  │
│             │  ── 容纳 ──      │
│             │  【栎树】【松树】  │
│             │                  │
└─────────────┴──────────────────┘
```

### 3.2 实现

- `bevy_ui` 的 `NodeBundle` + `TextBundle`
- 面板内容通过 Bevy System 每帧更新（仅在选中变化时重绘）
- 标签中文化：`TAG_ZH` 映射表

**参照**：Godot `selection_info_panel.gd` 的 `TAG_ZH` 字典。全量迁移中文标签。

### 3.3 容纳列表

- 选中树 → 面板显示"容纳："列表项（栎树/松树/鸟巢）
- 点击容纳列表中的项 → 切换到该项的详情
- 同 Godot `ui_containment_entries` 逻辑

## 四、拖拽

### 4.1 基本行为

- 鼠标按住卡 → 拖拽 → 松开放到目标格
- 拖拽时卡跟随鼠标
- 目标格有兼容卡 → 触发右键关系（叠加/组合）
- 目标格无兼容卡 → 放到空格
- 目标格不可放（水/边缘）→ 弹回原位

### 4.2 实现

- Bevy 输入系统：`MouseButtonInput` + `CursorMoved`
- 拖拽状态：`DragState` Resource
- 同 Godot `drag_manager.gd` 的左右键逻辑
