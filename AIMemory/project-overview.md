# 方寸商国：桃花源记 — 项目概述

> 给完全不了解本项目的人。

---

## 这是什么游戏

一个标签驱动的生态模拟游戏。南宋商人逃入桃花源，从零开始生存、建造、交易，最终把一个小山谷发展成商业枢纽。游戏的深度不在脚本，在**所有实体自己涌现行为**——像矮人要塞的猫集体醉酒一样不可预测。

---

## 和环世界、矮人要塞的对比

| | 环世界 | 矮人要塞 | 方寸商国 |
|---|---|---|---|
| 行为引擎 | 需求系统 + 工作分配 | 硬编码行为树 | **标签涌现——标签互碰产生行为** |
| 加新物种 | 写 C# 代码 | 写 token 文件 | 写几行标签配置 |
| 生态模拟 | 基础（温度、种植） | 深度（身体部位、液体） | **公理引擎——不硬写每条规则** |
| AI 智能 | 微操 + 全局规划 | 固定行为 | **效用打分 + 公告栏方向感** |
| 性能瓶颈 | 单线程 + 物件多 | 逻辑爆炸 | **实体上限约 5000，不碰天花板** |
| 创作自由 | 受限于代码架构 | 受限于 token 系统 | **标签等同"万能通行证"——恶魔也能做** |

## 核心架构优势

### 1. 标签 = 设计语言

不写代码。写标签。`狼 = predator + carnivore + pack_hunter`。羊 = `herbivore + flocking + flight_distance(6)`。新增一个物种 = 几行 RON 配置文件。设计师可以直接操作游戏逻辑。

### 2. 公理 = 自动纠错

四条引擎定律（compose/traverse/perceive/transform）自动处理"能不能进这格""能不能跨介质""能不能感知到""能量怎么流"。不需要为每个新物种写"狐狸不能下水"——公理替你挡住，标签替你放行。

### 3. 性能无后顾之忧

三道优化完成：spatial query 网格索引、实体休眠跳过空闲 tick、批量统一更新。在 5000 实体以内，一帧 15ms，不需要考虑性能。只考虑设计。

### 4. 涌现深度不被代码量约束

两卡碰撞 → 标签互碰 → 新状态涌现。200 张卡不是 200 种硬编码行为——是标签组合产生的指数级可能性。矮人要塞的猫醉酒是黑箱涌现，我们的系统能产生更复杂的链条。

### 5. 工作流自动化

DeepSeek + Cursor 双 AI 协作。设计文档、标签字典、机读断言三件套保证无歧义。GitHub CI 自动测试。不需要等编译结果——AI 自己查 BRP 验证。

## 发展极限

| 维度 | 极限 |
|---|---|
| 实体数 | ~5000（地图扩 6 倍后） |
| 帧率 | 15ms/tick @ 5000 实体 |
| 标签种类 | 无上限 |
| 公理条数 | 当前 4 条，可扩展 |
| 智能深度 | 效用引擎 + 公告栏 + 因果涌现 |
| 内容扩展 | 加卡 = 加标签，不加代码 |
