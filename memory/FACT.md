# 铁律

## 世界基本规则
- 游戏只有两种东西：格子（地形）和卡。卡一一对应放在格子上。
- 一格一卡，无例外。群卡已砍。
- 昆虫无实卡，纯容纳（in_pool/in_tree/in_ground）。包括 waterBug、landBug。
- 树、山、石头等 rooted 卡不可移动，不可被挤开。
- 曼哈顿移动：任何移动只能单轴（dx/dy 不同时非零）。

## 交互系统
- 左键砸（攻击+加工，红圈），右键叠（改变本质+搬运，半透明幽灵）。
- 1砸=1HP，2砸=出产品。不可连续蹭，必须拉开再碰。
- AI 攻击 = 砸，AI 搬运 = 幽灵。

## 架构铁律
- 标签是存在，公理处理交互。禁止物种硬编码（type_name 匹配）。
- compose 默认拒绝进入已占格，仅 corpses 和 incorporeal 例外。
- 每个 handoff 必须含架构计划、架构反馈、智能验收三段。
- 所有代码改动通过 handoff → Cursor，我不直接改项目文件。

## 工作流铁律
- 每次讨论收束时确认：哪些是铁律，哪些暂定，哪些放着。
- 新增/修改前必须读已有的 AIMemory/design_* 文件，避免前后矛盾。
- Cursor 提交前必须 cargo test 全 PASS + smoke PASS。
- 0 移动或 0 捕猎的 smoke 视为失败。
